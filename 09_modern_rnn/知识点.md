# 9.1 GRU（门控循环单元）
## 一、核心知识点总结
### 1. GRU核心定位
GRU是**简化版LSTM**，解决RNN梯度消失问题，**计算更快、参数更少**，效果与LSTM相当，擅长处理长序列依赖。

### 2. 核心解决问题
- 保留早期重要信息（如校验和）
- 跳过无关词元（如HTML冗余代码）
- 序列中断时重置状态（如章节切换、牛熊市切换）

### 3. 两大核心门控（0~1向量，sigmoid输出）
- **重置门Rₜ**：控制保留多少**过去状态**，负责**短期依赖**
  公式：$R_t = \sigma(X_t W_{xr} + H_{t-1} W_{hr} + b_r)$
- **更新门Zₜ**：控制新状态由**旧状态/候选状态**各占多少，负责**长期依赖**
  公式：$Z_t = \sigma(X_t W_{xz} + H_{t-1} W_{hz} + b_z)$

### 4. 三步计算流程
1. **候选隐状态**：结合重置门过滤旧信息
   公式：$\tilde{H}_t = \tanh(X_t W_{xh} + (R_t \odot H_{t-1}) W_{hh} + b_h)$
2. **最终隐状态**：更新门做加权融合
   公式：$H_t = Z_t \odot H_{t-1} + (1-Z_t) \odot \tilde{H}_t$
3. **输出**：$Y_t = H_t W_{hq} + b_q$

### 5. 门控关键作用
- Zₜ→1：保留旧状态，**跳过当前输入**，保护长期记忆
- Zₜ→0：用候选状态覆盖，**接受新信息**
- Rₜ→1：恢复普通RNN，关注短期
- Rₜ→0：清空旧状态，**重置记忆**

### 6. 优势
- 比LSTM少**1个门+1个细胞状态**，参数量小、速度快
- 有效缓解梯度消失，适配长序列

## 二、课后练习题答案
### 练习1：仅用t′时刻输入预测t>t′时刻输出，Rₜ/Zₜ最佳值
- **重置门Rₜ**：t≤t′时Rₜ=1（保留历史）；t>t′时Rₜ=0（完全重置，不依赖更早状态）
- **更新门Zₜ**：t≤t′时Zₜ=1（固定t′状态）；t>t′时Zₜ=0（只使用当前候选状态）
- 结论：**Rₜ=0，Zₜ=0**，只依赖当前输入，不依赖历史

### 练习2：超参数影响分析
- 隐藏单元num_hiddens↑：拟合能力↑、速度↓、过拟合风险↑
- 学习率lr↑：收敛快→不稳定；lr↓：收敛慢→更稳定
- 序列长度num_steps↑：长依赖更好→计算更慢
- 批次batch_size↑：梯度更稳→显存占用↑

### 练习3：RNN vs GRU对比
- 速度：GRU >> 普通RNN
- 困惑度：GRU更低（拟合更好）
- 输出：GRU更通顺、长程逻辑更连贯

### 练习4：只实现一个门会怎样
- 仅重置门：丢失长期记忆，退化为短期RNN，梯度消失复发
- 仅更新门：无法过滤短期噪声，状态易被无关信息污染

## 三、高频面试题+标准答案
### 1. GRU和普通RNN的核心区别？
- 普通RNN：无门控，梯度消失严重，长序列依赖差
- GRU：加入**重置门+更新门**，可学习保留/遗忘/更新，缓解梯度消失，长依赖更强

### 2. GRU和LSTM的区别？
| 对比项 | GRU | LSTM |
|--------|-----|------|
| 门数量 | 2个（重置、更新） | 3个（遗忘、输入、输出） |
| 状态数 | 1个隐状态 | 细胞状态+隐状态 |
| 参数量 | 更少 | 更多 |
| 速度 | 更快 | 较慢 |
| 效果 | 多数场景相当 | 复杂任务略优 |
| 适用场景 | 轻量、实时、长文本 | 极复杂序列 |

### 3. 重置门和更新门分别做什么？
- 重置门：控制**遗忘过去**，捕捉**短期依赖**
- 更新门：控制**保留过去/接受新信息**，捕捉**长期依赖**

### 4. GRU如何缓解梯度消失？
- 更新门可让$H_t≈H_{t-1}$，梯度近似1，避免连乘衰减
- 门控自适应调整记忆，减少无效梯度传播

### 5. Zₜ=1/Zₜ=0、Rₜ=1/Rₜ=0分别代表什么？
- Zₜ=1：完全保留旧状态，忽略当前输入
- Zₜ=0：完全用新候选状态
- Rₜ=1：保留全部旧状态，等价普通RNN
- Rₜ=0：清空旧状态，只看当前输入

### 6. 为什么GRU用sigmoid和tanh？
- sigmoid：输出0~1，适合做**门控权重**
- tanh：输出-1~1，保证**隐状态数值稳定**，梯度更友好

### 7. GRU在NLP的典型应用？
- 文本生成、情感分析、机器翻译、语音识别、时间序列预测

### 8. GRU的前向传播公式默写（高频）
1. $R_t = \sigma(X_t W_{xr} + H_{t-1} W_{hr} + b_r)$
2. $Z_t = \sigma(X_t W_{xz} + H_{t-1} W_{hz} + b_z)$
3. $\tilde{H}_t = \tanh(X_t W_{xh} + (R_t \odot H_{t-1}) W_{hh} + b_h)$
4. $H_t = Z_t \odot H_{t-1} + (1-Z_t) \odot \tilde{H}_t$

# 9.2 LSTM（长短期记忆网络）

# 一、LSTM 核心知识点总结
## 1. LSTM 定位
LSTM 是**最早解决 RNN 梯度消失**的门控 RNN，比 GRU 早 20 年，结构更复杂、记忆能力更强，适合**极长序列依赖**。

## 2. 核心创新：记忆元 Cell
- 新增**内部记忆元 Cₜ**，专门负责**长期记忆存储**
- 隐状态 Hₜ 只负责输出，不负责存储长期信息
- 用**三个门**精确控制记忆的**遗忘、写入、输出**

## 3. 三大核心门（sigmoid 输出 0~1）
1. **遗忘门 Fₜ**：控制**保留多少旧记忆 Cₜ₋₁**
   Fₜ = σ(XₜWₓբ + Hₜ₋₁Wₕբ + bբ)
2. **输入门 Iₜ**：控制**写入多少新候选记忆**
   Iₜ = σ(XₜWₓᵢ + Hₜ₋₁Wₕᵢ + bᵢ)
3. **输出门 Oₜ**：控制**从记忆中输出多少到隐状态**
   Oₜ = σ(XₜWₓₒ + Hₜ₋₁Wₕₒ + bₒ)

## 4. 候选记忆元 & 记忆元更新
- **候选记忆**：$\tilde{C}_t = \tanh(X_tW_{xc}+H_{t-1}W_{hc}+b_c)$
- **最终记忆**：$C_t = F_t \odot C_{t-1} + I_t \odot \tilde{C}_t$
  - Fₜ→1：长期记忆原样保留
  - Iₜ→1：写入新信息

## 5. 隐状态计算
$H_t = O_t \odot \tanh(C_t)$
- 输出门控制是否把记忆“暴露”给下一层
- tanh 保证数值稳定在 (-1,1)

## 6. LSTM 优势
- 完美解决**长序列梯度消失**
- 记忆可长期保存，适合文本、语音、时间序列
- 比 GRU 表达能力更强，适合复杂任务

## 练习1：超参数对运行时间、困惑度、输出的影响
- **num_hiddens↑**：拟合更好→困惑度↓；计算量↑→速度↓；易过拟合
- **lr↑**：收敛快→震荡；lr↓：收敛稳→慢
- **num_steps↑**：长依赖更好→速度↓
- **batch_size↑**：梯度更稳→显存占用↑

## 练习2：如何生成单词而非字符序列
- 把**字符级 token 换成单词级 token**
- 使用分词工具（jieba/BPE）构建词表
- 输入输出均为单词索引

## 练习3：GRU / LSTM / RNN 计算成本对比
- 普通 RNN：成本最低，速度最快
- GRU：2个门，成本中等，速度次之
- LSTM：3个门 + 记忆元，**参数量最大、计算成本最高**
- 训练/推理成本：**RNN < GRU < LSTM**

## 练习4：候选记忆已用 tanh，为何隐状态还要 tanh
- 候选记忆 tanh 是为了**稳定写入记忆**
- 隐状态再 tanh 是为了**输出值域标准化**，保证梯度稳定
- 输出门 Oₜ 是 0~1，与 tanh(Cₜ) 相乘后，Hₜ 仍在 (-1,1)

## 练习5：LSTM 做时间序列预测（改法）
- 输入：连续时间步特征
- 输出：下一时刻值（回归任务）
- 损失函数：MSE 替代交叉熵
- 输出层激活：去掉 softmax，用线性输出

# 三、高频面试题 + 标准答案
## 1. LSTM 解决了什么问题？
- 解决普通 RNN **梯度消失**，无法捕获长距离依赖的问题。
- 通过**记忆元 + 三门**实现长期信息可控保存。

## 2. LSTM 三门分别干什么？
- **遗忘门**：决定丢弃多少旧记忆
- **输入门**：决定写入多少新信息
- **输出门**：决定把多少记忆输出给隐状态

## 3. LSTM 和 GRU 的区别（必考）
| 项 | LSTM | GRU |
|----|------|-----|
| 门数 | 3个 | 2个 |
| 状态 | C（记忆）+ H（隐态） | 只有 H |
| 参数 | 更多 | 更少 |
| 速度 | 更慢 | 更快 |
| 表达力 | 更强 | 足够强 |
| 场景 | 复杂长序列 | 轻量、实时 |

## 4. LSTM 如何缓解梯度消失？
- 记忆元 Cₜ = FₜCₜ₋₁ + Iₜ$\tilde{C}_t$
- 当 Fₜ≈1 时，Cₜ≈Cₜ₋₁，**梯度近似 1 直接传递**，不会连乘衰减。

## 5. Cₜ 和 Hₜ 的区别？
- **Cₜ 记忆元**：内部长期存储，不直接输出
- **Hₜ 隐状态**：对外输出，由 Cₜ 经过输出门与 tanh 得到

## 6. 为什么用 sigmoid 和 tanh？
- sigmoid：输出 0~1，适合做**门控权重**
- tanh：输出 -1~1，保证**数值稳定、梯度友好**

## 7. 如果 Fₜ=1、Iₜ=0 会怎样？
- 记忆元 Cₜ = Cₜ₋₁，**信息永久保存**，完全不更新，完美保留早期信息。

## 8. LSTM 前向传播公式（默写版）
1. Fₜ = σ(XₜWₓբ + Hₜ₋₁Wₕբ + bբ)
2. Iₜ = σ(XₜWₓᵢ + Hₜ₋₁Wₕᵢ + bᵢ)
3. Oₜ = σ(XₜWₓₒ + Hₜ₋₁Wₕₒ + bₒ)
4. $\tilde{C}_t = \tanh(X_tW_{xc}+H_{t-1}W_{hc}+b_c)$
5. Cₜ = Fₜ⊙Cₜ₋₁ + Iₜ⊙$\tilde{C}_t$
6. Hₜ = Oₜ⊙tanh(Cₜ)

## 9. LSTM 适用场景
- 文本生成、机器翻译、情感分析、语音识别、时间序列预测、命名实体识别

# 9.3  深度循环神经网络（Deep RNN）
## 一、核心一句话总结
**深度循环神经网络（Deep RNN）= 把多个单层 RNN/LSTM/GRU 垂直堆叠起来**，上层隐状态由下层输出与自身上一时刻隐状态共同计算，实现**更高级的序列特征抽象**，拟合更复杂的时序依赖。

---

## 二、为什么需要 Deep RNN？
- 单层 RNN 表达能力有限，只能学习**浅层时序特征**
- 深层结构可分层提取特征：**低层抓局部时序模式，高层抓全局语义/长程依赖**
- 同样参数量下，**深度 > 宽度**，特征表达更强
- 适配 NLP、语音、时间序列等复杂序列任务

---

## 三、数学定义与前向传播（严格对齐 D2L）
设：
- 时间步 t 输入：**Hₜ⁽⁰⁾ = Xₜ**（n 个样本，d 维特征）
- 共 L 层隐藏层，第 l 层隐状态：Hₜ⁽ˡ⁾ ∈ ℝⁿˣʰ
- 激活函数 φ（如 tanh、ReLU）

### 1. 第 l 层更新公式
Hₜ⁽ˡ⁾ = φ( Hₜ⁽ˡ⁻¹⁾ Wₓₕ⁽ˡ⁾ + Hₜ₋₁⁽ˡ⁾ Wₕₕ⁽ˡ⁾ + bₕ⁽ˡ⁾ )

### 2. 输出层
Yₜ = Hₜ⁽ᴸ⁾ Wₕq + b_q

### 3. 关键理解
- **垂直方向**：第 l 层输入 = 第 l-1 层当前时刻输出
- **时间方向**：每层内部**时间步间共享权重**
- **整体**：时间维度循环 + 层间前馈，构成**深度循环结构**

---

## 四、直观结构流程
1. 输入 Xₜ → 第一层 RNN → 得到 Hₜ⁽¹⁾
2. Hₜ⁽¹⁾ → 第二层 RNN → 得到 Hₜ⁽²⁾
3. ……
4. 第 L 层输出 Hₜ⁽ᴸ⁾ → 全连接层 → 最终输出 Yₜ

每层都保留**自己的上一时刻隐状态**，只把**当前时刻输出传给下一层**。

---

## 五、参数量与计算特点
- **参数共享**：同一层、不同时间步权重完全共享
- **层数 L 增加**：参数量近似线性增加，不会爆炸
- **计算代价**：时间步必须串行，**无法并行**，深层训练更慢
- **梯度问题**：层数变多，**梯度消失/爆炸更严重**，常用 LSTM/GRU + 梯度裁剪

---

## 六、D2L 简洁实现（PyTorch）
核心：用 **num_layers** 控制深度
```python
import torch
from torch import nn

# 构造 Deep RNN
num_layers = 2  # 2层深度循环
rnn = nn.RNN(
    input_size=input_size,
    hidden_size=num_hiddens,
    num_layers=num_layers,  # 关键参数
    batch_first=True
)

# 前向
# inputs: (batch_size, seq_len, input_size)
outputs, h_n = rnn(inputs)
# outputs: (batch_size, seq_len, num_hiddens) 所有时间步最后一层输出
# h_n: (num_layers, batch_size, num_hiddens) 每一层最后时刻隐状态
```

## 七、D2L 本节课后题（原题+标准答案）
### 题目1：如果使用深层循环神经网络，是否会导致参数数量急剧增加？为什么？
**答**：不会急剧增加。
同一层内**时间步之间权重共享**，只有层间参数独立。参数量随层数**近似线性增长**，而非指数增长。

### 题目2：尝试构建一个多层双向循环神经网络
**答**：同时设置 bidirectional=True 与 num_layers：
```python
nn.RNN(
    input_size, hidden_size,
    num_layers=2,
    bidirectional=True,
    batch_first=True
)
```
此时隐状态维度变为 **2*hidden_size**（前向+后向拼接）。

### 题目3：深度 RNN 与深度前馈网络的核心区别是什么？
**答**：
- 深度 RNN：**时间维度循环 + 层间垂直堆叠**，处理序列，时间步串行
- 深度前馈网络：纯垂直层叠，无时间循环，处理固定维度向量，可全并行

### 题目4：为什么深层 RNN 常搭配 LSTM/GRU 使用？
**答**：层数加深会加剧**梯度消失/爆炸**。LSTM/GRU 门控机制可稳定梯度，保证深层能学到长程依赖。

## 八、高频面试题（大厂必考+满分答案）
### 1. 什么是 Deep RNN？和单层 RNN 区别？
**答**：
Deep RNN 是**多层 RNN 垂直堆叠**的循环结构。
区别：
- 单层：仅一层循环，学习浅层时序特征
- 深层：多层循环，分层提取特征，表达能力更强，可建模更复杂依赖

### 2. Deep RNN 如何进行前向传播？
**答**：
1. 输入 Xₜ 进入第一层
2. 第 l 层用**下层当前输出 + 自身上一时刻隐状态**更新
3. 逐层向上计算，最后一层输出接全连接层得到预测

### 3. Deep RNN 训练难点 & 解决办法？
**答**：
难点：
- 梯度消失/爆炸更严重
- 时间步串行，训练慢
解决：
- 用 LSTM/GRU 替代普通 RNN
- 梯度裁剪（clip gradient）
- 合适初始化 + 层归一化
- 用 Transformer 并行建模长序列

### 4. 层数 L 越多越好吗？如何选？
**答**：不是。
层数过多会**梯度恶化、过拟合、训练极慢**。
经验：
- 文本：2~4 层
- 语音：3~6 层
优先用**双向 LSTM/GRU**，再适当加深。

### 5. Deep RNN 与 Bidirectional RNN 关系？
**答**：正交设计，可叠加使用。
- Deep：**垂直堆叠层数**
- Bidirectional：**同时利用过去+未来信息**
工业界常用：**2~3层双向LSTM**




#  9.4 双向循环神经网络（Bi-RNN）
**双向循环神经网络 = 前向RNN（从左→右）+ 反向RNN（从右→左）**，同一时间步同时使用**过去 + 未来**全部上下文，专门解决需要完整语境的序列任务。

---

## 为什么需要双向RNN？
- 单向RNN只能用**过去信息**，看不到下文
- 填空、命名实体识别、词义消歧必须依赖**前后文**
- 类比隐马尔可夫模型的**前向-后向算法**，双向结构天然适合序列插值
- 能捕捉完整序列依赖，显著提升需要全局上下文的任务精度

---

## 三、数学定义（严格对齐D2L公式）
设：
- 时间步 t 输入：$X_t$
- 前向隐状态：$\overrightarrow{H}_t$
- 反向隐状态：$\overleftarrow{H}_t$
- 激活函数 $\phi$（tanh/ReLU）

1. **前向隐状态更新**（从过去到现在）
$$\overrightarrow{H}_t = \phi(X_t W_{xh}^{(f)} + \overrightarrow{H}_{t-1} W_{hh}^{(f)} + b_h^{(f)})$$

2. **反向隐状态更新**（从未来到现在）
$$\overleftarrow{H}_t = \phi(X_t W_{xh}^{(b)} + \overleftarrow{H}_{t+1} W_{hh}^{(b)} + b_h^{(b)})$$

3. **拼接得到最终隐状态**
$$H_t = [\overrightarrow{H}_t, \overleftarrow{H}_t]$$
维度变为 **2 × 隐藏单元数**

4. **输出层**
$$O_t = H_t W_{hq} + b_q$$

---

## 四、结构与计算特点
- **方向独立**：前向、反向权重不共享，各自学习时序模式
- **信息完整**：每个时间步都能看到**整个序列的上下文**
- **维度翻倍**：隐状态维度变为 2h
- **计算更慢**：前向需要正反两次遍历，梯度链更长，训练代价高

---

## 五、适用场景 vs 绝对不能用的场景
**适合（必须用双向）**
- 文本填空、完形填空
- 命名实体识别（NER）
- 词性标注、句法分析
- 机器翻译编码器
- 词义消歧（一词多义）
- 情感分类（需全句理解）

**禁止使用（会完全失效）**
- 语言模型、下一个词预测
- 实时流式推理（未来信息不可达）
- 时序预测（股价、流量预测）

# 七、D2L 本节课后题（原题+标准答案）
### 题1：如果不同方向使用不同数量的隐藏单元，$H_t$ 的形状会怎样变化？
**答**：
假设前向隐藏数 h₁，反向隐藏数 h₂，
则拼接后 $H_t$ 的维度为 **h₁ + h₂**，不再是 2h。
输出层权重 $W_{hq}$ 输入维度也要对应改为 h₁+h₂。

---

### 题2：设计一个具有多个隐藏层的双向循环神经网络
**答**：
设置 `num_layers ≥ 2` 且 `bidirectional=True` 即可：
```python
nn.LSTM(
    input_size=100,
    hidden_size=256,
    num_layers=2,  # 多层
    bidirectional=True,
    batch_first=True
)
```
每一层都是双向结构，上层输入是下层双向拼接后的输出。

---

### 题3：一词多义如何用神经网络处理？哪种架构最合适？
**答**：
1. **模型设计**
   - 输入：完整句子序列
   - 用 **双向RNN/LSTM/GRU** 或 **BERT** 编码
   - 取目标词对应的双向隐状态作为上下文向量
2. **最优架构**
   - **双向RNN/LSTM**（经典方案）
   - **Transformer/BERT**（当前SOTA）
   两者都能利用**完整上下文**消除歧义。

# 八、高频面试题（大厂必考+满分答案）
### 1. 什么是双向RNN？和单向RNN的区别？
**答**：
双向RNN同时训练**前向+反向**两个RNN，每个时间步拼接两个方向隐状态。
- 单向：只用过去信息
- 双向：同时用**过去 + 未来**上下文

---

### 2. 双向RNN为什么不能做语言模型/下一词预测？
**答**：
下一词预测时**未来信息不可达**，而双向RNN在训练时偷看了未来文本，
测试时无未来信息，精度会急剧下降，完全不可用。

---

### 3. 双向RNN的前向传播流程？
**答**：
1. 从左到右计算前向隐状态 $\overrightarrow{H}_t$
2. 从右到左计算反向隐状态 $\overleftarrow{H}_t$
3. 每个时间步拼接两个隐状态 $H_t = [\overrightarrow{H}_t, \overleftarrow{H}_t]$
4. 送入输出层得到结果

---

### 4. 双向RNN的优缺点？
**答**
优点：
- 利用完整上下文，表达能力强
- 适合NLP标注类、理解类任务
缺点：
- 计算慢，训练代价高
- 不能用于流式/预测任务
- 梯度链更长，更易梯度消失

---

### 5. 深度双向RNN是什么？
**答**：
把**多层双向RNN垂直堆叠**，每一层都是双向结构，
上层输入是下层双向拼接后的隐状态，常用于机器翻译、语音识别。

# 一、9.6 编码器-解码器架构
## 1. 核心定位
编码器-解码器是**处理变长输入→变长输出**的序列转换基础架构，是机器翻译、文本摘要、对话生成等任务的**通用骨架**。

## 2. 整体流程
1.  **编码器**：接收变长输入序列 → 压缩为**固定维度语义状态**（上下文向量）。
2.  **解码器**：接收语义状态 → 自回归**逐词生成**变长输出序列。
3.  **整体**：Encoder 负责“理解输入”，Decoder 负责“生成输出”，通过**中间状态**连接。

## 3. 三大组件接口（代码核心）
### （1）Encoder 基类
- 输入：变长序列 X
- 输出：编码结果（用于构造解码器初始状态）
- 核心方法：`forward`（前向编码）

### （2）Decoder 基类
- 新增：`init_state`（把 Encoder 输出转为 Decoder 初始状态）
- 输入：当前时间步输入 + 上一状态
- 输出：当前时间步预测 + 新状态
- 核心：**自回归生成**

### （3）EncoderDecoder 合并类
- 组合 Encoder + Decoder
- 前向逻辑：
  enc_X → Encoder → enc_outputs → init_state → Decoder(dec_X, state) → 输出序列

## 4. 关键特点
- 输入输出**长度可不同**
- 编码器只做**压缩理解**，解码器只做**生成**
- 是 seq2seq、Transformer 等模型的**底层架构**

---

# 二、课后练习题标准答案
### 练习1：编码器和解码器必须是同一类型神经网络吗？
**答案：不必相同。**
- 编码器可用 CNN、RNN、Transformer Encoder、MLP 等任意能输出固定维度表征的网络。
- 解码器可用 RNN、Transformer Decoder、GRU、LSTM 等任意自回归生成网络。
- 只需满足：**编码器输出能通过 init_state 转为解码器合法初始状态**（维度/结构匹配）即可。

### 练习2：除机器翻译外，还有哪些适用场景？
**答案：所有“输入序列→输出序列”的任务**，典型包括：
1. 文本摘要（长文本→短摘要）
2. 对话系统（用户问句→机器人回复）
3. 语音识别（语音序列→文字序列）
4. 语音合成（文本→语音帧序列）
5. 代码生成（自然语言→代码）
6. 句式改写/语法纠错
7. 图像描述（图像特征序列→文本）

# 三、高频面试题 + 标准答案
### 1. 什么是编码器-解码器架构？核心作用是什么？
**答**：
由编码器和解码器两部分组成，用于**变长序列到变长序列**的转换任务。
- 编码器：把输入序列编码为固定维度**上下文向量**，完成语义理解。
- 解码器：基于上下文向量**自回归生成**输出序列。
- 作用：统一处理输入输出长度不一致的序列问题，是 NLP 生成式模型基础。

### 2. Encoder 与 Decoder 的核心区别是什么？
| 模块 | 功能 | 信息流向 | 典型任务 |
|---|---|---|---|
| Encoder | 理解、压缩输入 | 双向/全局 | 分类、NER、语义匹配 |
| Decoder | 生成、预测输出 | 单向/自回归 | 翻译、摘要、对话 |

### 3. 为什么解码器要用自回归生成？
**答**：
保证生成**时序合法性**：每一步只依赖**已生成的前缀**，不看到未来 token，避免信息泄露，保证推理与训练一致。

### 4. 编码器输出的“状态”在架构中起什么作用？
**答**：
- 作为**上下文桥梁**，把输入的全局语义传递给解码器。
- 是解码器的**初始隐状态**，决定生成的起点与整体语义方向。
- 控制信息压缩程度，影响生成质量。

### 5. 只编码器/只解码器/编码器-解码器三种架构分别适合什么任务？
**答**：
- **Encoder-only**（如 BERT）：双向注意力，适合**理解类**：文本分类、NER、相似度匹配。
- **Decoder-only**（如 GPT）：因果掩码，适合**自回归生成**：对话、续写、代码补全。
- **Encoder-Decoder**（如 Transformer、T5）：理解+生成，适合**序列转换**：翻译、摘要、多轮对话。

### 6. 编码器-解码器的主要缺点是什么？怎么改进？
**答**：
- 缺点：编码器把**整段输入压成一个向量**，长序列时**信息丢失/梯度消失**，后面词覆盖前面词。
- 改进：引入**注意力机制**，让解码器每一步都能直接关注输入序列任意位置，解决长程依赖问题。

### 7. 编码器和解码器如何连接？init_state 作用是什么？
**答**：
- 连接：Encoder 输出 → `init_state` → Decoder 初始状态。
- `init_state`：把编码器输出**适配**成解码器能接受的状态格式（维度、层数、结构对齐）。


# seq2seq（序列到序列学习）超详细讲解（含习题+面试题答案）

---

## 9.7 seq2seq 核心原理（
**seq2seq = 编码器-解码器架构 + 循环神经网络，专门解决「输入序列、输出序列长度都可变」的序列生成问题**，最经典场景：机器翻译。

---

## 二、整体架构（Encoder-Decoder）
### 1. 核心流程
1. **编码器**：把变长输入序列 → 压缩成**固定维度上下文向量 c**（把整句信息装进一个向量）。
2. **解码器**：以 c 为起点，逐词生成输出序列，直到输出 `<eos>` 停止。
3. 特殊符号：
   - `<bos>`：序列开始，作为解码器第 1 个输入
   - `<eos>`：序列结束，生成即停止
   - `<pad>`：填充，让不同长度序列能批处理

### 2. 数学表达
- 编码器：
  \[
  h_t = f(x_t, h_{t-1}),\quad c = q(h_1,...,h_T)
  \]
  常用取最后时刻隐状态：**c = h_T**。
- 解码器：
  \[
  s_{t'} = g(y_{t'-1}, c, s_{t'-1})
  \]
  用 s 预测当前词概率分布。

---

## 三、编码器（Encoder）详解
### 1. 作用
- 输入：源语言序列（如英文句子）
- 输出：所有时刻隐状态 + **最后时刻隐状态**（用来初始化解码器）

### 2. 结构
嵌入层 → 多层 GRU/LSTM → 输出隐状态序列与最终隐状态。

### 3. 代码逻辑（PyTorch 风格）
```python
class Seq2SeqEncoder(nn.Module):
    def __init__(self, vocab_size, embed_size, num_hiddens, num_layers, dropout=0):
        self.embedding = nn.Embedding(vocab_size, embed_size)
        self.rnn = nn.GRU(embed_size, num_hiddens, num_layers, dropout=dropout)
    def forward(self, X):
        X = self.embedding(X)       # (batch, seq_len, embed)
        X = X.permute(1, 0, 2)     # 转成 (seq_len, batch, embed)
        output, state = self.rnn(X)
        return output, state       # state 用于解码器初始化
```
- output 形状：(seq_len, batch, num_hiddens)
- state 形状：(num_layers, batch, num_hiddens)

---

## 四、解码器（Decoder）详解
### 1. 作用
- 输入：`<bos>` + 已生成的词 + 上下文 c
- 输出：下一个词的概率分布

### 2. 关键设计
1. 用编码器**最后隐状态**初始化解码器隐状态。
2. 每个时间步把 c 与输入拼接，让解码器全程看到全局信息。
3. 最后接全连接层 + softmax 输出词表概率。

### 3. 代码逻辑
```python
class Seq2SeqDecoder(nn.Module):
    def __init__(self, vocab_size, embed_size, num_hiddens, num_layers, dropout=0):
        self.embedding = nn.Embedding(vocab_size, embed_size)
        self.rnn = nn.GRU(embed_size + num_hiddens, num_hiddens, num_layers, dropout=dropout)
        self.dense = nn.Linear(num_hiddens, vocab_size)
    def init_state(self, enc_outputs):
        return enc_outputs[1]  # 用编码器最终 state 初始化
    def forward(self, X, state):
        X = self.embedding(X).permute(1,0,2)
        context = state[-1].unsqueeze(0).repeat(X.shape[0], 1, 1)
        X = torch.cat((X, context), dim=-1)  # 输入 + 上下文拼接
        output, state = self.rnn(X, state)
        output = self.dense(output).permute(1,0,2)
        return output, state
```

## 五、损失函数与 Mask
### 1. 为什么要用 sequence_mask？
- 序列长度不一，用 `<pad>` 补齐到相同长度
- 若不屏蔽填充位，模型会**学习去预测 `<pad>`**，严重降低效果。

### 2. 遮蔽效果
把有效长度外的位置置 0，不参与损失计算：
```
[[1,2,3]] → 有效长度 1 → [[1,0,0]]
[[4,5,6]] → 有效长度 2 → [[4,5,0]]
```

## 六、d2l 9.7 课后习题 + 标准答案
### 习题 1：调整超参数改善翻译效果
- 答案：增大 embed_size、num_hiddens、num_layers，适当调小 lr、batch_size，增加 epoch，搭配合理 dropout，可提升 BLEU 与翻译准确率。

### 习题 2：不使用遮蔽（mask）会怎样？
- 现象：训练不稳定、生成大量重复无意义词、BLEU 暴跌。
- 原因：模型学习拟合 `<pad>` 填充位，有效词梯度被干扰，损失被无效位置污染。

### 习题 3：编码器和解码器层数/隐藏单元数不同，如何初始化？
- 答案：
  1. 层数不匹配：编码器层数 > 解码器，取最后几层；反之补零层
  2. 维度不匹配：加**线性层**把编码器隐状态映射到解码器维度
  \[
  s_0 = Linear(h_T)
  \]


### 习题 4：不用 Teacher Forcing，用前一步预测输入，有何影响？
- 答案：
  - 训练接近推理真实场景，泛化可能略好
  - 但**误差累积**严重，训练慢、难收敛、性能大幅下降
  - 实际常用：Teacher Forcing + 偶尔采样（Schedule Sampling）

### 习题 5：用 LSTM 替换 GRU
- 答案：
  - 结构几乎不变，仅把 nn.GRU 换成 nn.LSTM
  - state 变为 (h, c)，初始化时要传两个状态
  - 效果相近，LSTM 稍稳但参数量更大、速度略慢

### 习题 6：解码器输出层的其他设计
- 答案：
  - 加多层感知机（2~3 层线性+激活）
  - 加残差连接、层归一化
  - 加注意力（后续章节）
  - 用自适应 softmax 处理大词表

## 七、seq2seq 高频面试题 + 标准答案
### 1. 什么是 seq2seq？典型应用有哪些？
- 答案：seq2seq 是 Encoder-Decoder 架构，把**一个变长序列映射到另一个变长序列**。
  应用：机器翻译、对话生成、文本摘要、语音识别、代码生成、视频描述。

### 2. seq2seq 编码器和解码器各自的作用？
- 答案：
  - 编码器：把输入序列编码为**固定维度上下文向量 c**，浓缩全局信息
  - 解码器：以 c 为条件，**自回归逐词生成**输出序列

### 3. 为什么解码器要用 Teacher Forcing？
- 答案：
  - 训练时直接喂 Ground Truth，避免误差累积
  - 收敛更快、训练更稳定
  - 缺点：训练与推理存在分布差异

### 4. seq2seq 的主要缺点是什么？
- 答案：
  1. 上下文向量 c 容量有限，**长序列信息丢失严重**
  2. 编码器和解码器都是 RNN 类，**无法并行**
  3. 长距离依赖建模弱

### 5. 怎么解决 seq2seq 长序列性能差的问题？
- 答案：
  1. 加**注意力机制**：每步动态计算上下文，不再只用固定 c
  2. 改用 Transformer：完全自注意力，并行+长距离能力极强

### 6. 训练和推理阶段的生成方式有何不同？
- 答案：
  - 训练：Teacher Forcing，一次喂整句，并行算损失
  - 推理：自回归生成，上一步输出当下一步输入，串行执行

### 7. seq2seq 与 Transformer 的关系？
- 答案：
  - seq2seq 是框架，RNN 是早期实现
  - Transformer 是**基于自注意力的 seq2seq 现代实现**，完全替代 RNN 版 seq2seq

---

## 八、总结（必背）
- seq2seq = 编码压缩 + 解码生成，解决**输入输出都变长**的序列问题
- 核心：编码器最后隐状态初始化解码器，上下文 c 贯穿解码全过程
- 训练关键：**mask 屏蔽填充**、**Teacher Forcing**
- 痛点：长序列信息丢失、串行慢 → 解决方案：**注意力 + Transformer**

# 一、BLEU（Bilingual Evaluation Understudy）
**BLEU 是机器翻译（seq2seq 生成）最常用的自动评价指标**，用来衡量**模型生成句子**和**参考标准答案（人工翻译）**的相似度。
值越高（0~1 或 0~100），翻译质量越好。

---

# 二、核心思想
BLEU 不看语法、不看语义，只看：
**生成的句子里，有多少 n-gram 出现在参考句子里**。

n-gram 指连续 n 个词：
- 1-gram：单词级别
- 2-gram：词组级别
- 3-gram、4-gram 更能反映流畅度

---

# 三、计算公式（简化版）
## 1. 基础精度（n-gram precision）
\[
P_n = \frac{\text{生成句子中出现在参考句中的 n-gram 总数}}{\text{生成句子中 n-gram 总数}}
\]

## 2. 短句惩罚（Brevity Penalty, BP）
机器翻译容易**生成过短句子**来刷高精度，所以要惩罚：
\[
BP =
\begin{cases}
1 & \text{若生成句长度 ≥ 参考句长度}\\
e^{1 - \frac{\text{参考长度}}{\text{生成长度}}} & \text{否则}
\end{cases}
\]

## 3. 最终 BLEU
通常用 **BLEU-4**（最标准）：
\[
BLEU = BP \cdot \exp\left(\frac{1}{4}\sum_{n=1}^4 w_n \log P_n\right)
\]
一般 \(w_n = 1/4\)，均匀加权。

---

# 四、通俗例子
参考翻译：A B C D
模型生成：A B B X

- 1-gram 命中：A、B → P1=2/4
- 2-gram 命中：AB → P2=1/3
- 句子太短 → BP < 1
→ BLEU 偏低

---

# 六、BLEU 相关面试题（高频）
## 1. BLEU 是用来评价什么任务的？
**答案**：
机器翻译、文本生成、对话生成、摘要生成等 seq2seq 任务。

## 2. BLEU 为什么要加 brevity penalty？
**答案**：
模型可能故意生成很短的句子，让 n-gram 精度虚高。BP 惩罚过短输出，保证评价公平。

## 3. BLEU 越高越好吗？有什么缺点？
**答案**：
- 越高表示与参考句越接近，通常越好。
- 缺点：
  - 不考虑语义、语法
  - 不考虑语序合理性
  - 依赖参考句质量
  - 对长文本不稳定

## 4. 为什么常用 BLEU-4 而不是更高阶？
**答案**：
n 太大（如 5/6）会导致几乎没有匹配，统计不可靠。4-gram 足够反映流畅度与准确性。

## 5. BLEU 能不能用于分类任务？
**答案**：
不能。BLEU 是**序列匹配指标**，分类用 accuracy、F1、AUC 等。

# 七、d2l 本节原书课后习题完整版（含答案）
## 习题 1：如果不使用 mask 会怎样？
**答案**：
填充符 `<pad>` 会被当作正常 token 训练，模型会倾向于生成大量 `<pad>`，损失函数被无效位置主导，翻译质量严重下降。

## 习题 2：如果编码器和解码器隐藏层维度不同，如何处理？
**答案**：
在编码器最终隐状态后加一个线性层，映射到解码器维度：
\[
s_0 = \text{Linear}(h_T)
\]

## 习题 3：如果编码器和解码器层数不同？
**答案**：
- 编码器层数更多：取最后几层
- 解码器层数更多：用全零/重复填充

## 习题 4：不使用 teacher forcing 会怎样？
**答案**：
训练时用前一步预测值作为输入，会导致**误差累积**，收敛慢、不稳定，翻译效果大幅下降。

## 习题 5：用 LSTM 替换 GRU 有什么变化？
**答案**：
结构基本不变，只是 state 从单一隐状态变成 (h, c)，初始化解码器时要同时传入两个状态。

## 习题 6：BLEU 指标在本节的作用？
**答案**：
量化评估翻译结果与标准答案的相似度，是训练过程中判断模型性能的核心指标。

# 八、seq2seq + BLEU 总结（面试必背）
- seq2seq = Encoder-Decoder 架构，处理变长→变长序列
- 训练关键：mask、teacher forcing
- 评价翻译质量用 **BLEU**
- BLEU = n-gram 精度 + 短句惩罚
- BLEU 越高，生成越接近参考句
- seq2seq 最大缺陷：长序列信息丢失 → 靠注意力机制解决




#  9.8 束搜索 Beam Search

## 1. 贪心搜索 Greedy Search
- **做法**：每一步只选**当前概率最大**的一个词，直到 <eos> 或最大长度
- **计算复杂度**：O(T′·|Y|)
- **优点**：极快、开销最小
- **缺点**：**局部最优≠全局最优**。某一步选次优词，后面整体概率可能更高
- **本质**：束宽 k=1 的束搜索

---

## 2. 穷举搜索 Exhaustive Search
- **做法**：枚举**所有可能的序列**，计算每条概率，选最大
- **复杂度**：O(|Y|^T′)
- **优点**：理论上能得到**全局最优**
- **缺点**：计算量爆炸，现实完全无法使用

---

## 3 束搜索 Beam Search（本节核心）
### （1）核心思想
- 每一步保留**概率最高的 k 个候选序列**，k 叫**束宽 beam size**
- 平衡效果与速度，介于贪心和穷举之间

### （2）执行步骤（k=2 示例）
1. 时间步 1：选概率最高的 **k 个词**作为 k 条候选起点
2. 后续每一步：
   - 对每条候选，扩展词表所有词，得到 **k×|Y|** 条新序列
   - 按**整条序列概率**排序，只保留前 k 条
3. 终止：遇到 <eos> 或达到最大长度
4. 最终打分选最优：
   \[
   \text{score} = \frac{1}{L^\alpha} \sum \log P(y_{t'}|y_{1..t'-1},c)
   \]
   - L：序列长度
   - α≈0.75：**惩罚长序列**（log 概率为负，长序列会被拉低分数）
5. 输出 score 最高的序列

### （3）复杂度
O(k·|Y|·T′)，k 越大越准、越慢；k 越小越快、越粗糙

### （4）关键结论
- 束搜索**只用于推理，不参与训练**，无参数可学
- 贪心 = k=1；k→∞ 近似穷举
- 必须做**长度归一化**，否则模型倾向生成短句子

---

## 5. 常见疑问（教材评论区高频）
1. **为什么不用维特比 Viterbi？**
   - 自然语言不满足严格马尔可夫性
   - 维特比复杂度 O(T·|Y|²)，词表大时开销爆炸
2. **Beam Search 需要训练吗？**
   不需要，是解码策略，模型训练完再用
3. **长度归一化为什么惩罚长序列？**
   log 概率都是负数，长序列求和更“负”，归一化后避免偏向短序列

# 二、课后习题 标准答案（d2l 9.8.5）
## 习题1：我们可以把穷举搜索看作一种特殊的束搜索吗？为什么？
**答案**：
可以。
当**束宽 k = 词表大小|Y|的序列长度次方**（即保留所有可能序列），束搜索等价于穷举搜索。
简单说：**k→∞ 时，束搜索 = 穷举搜索**。

---

## 习题2：在机器翻译中用束搜索，束宽如何影响预测速度和结果？
**答案**：
- **束宽 k 越大**
  - 结果越好：保留更多候选，更接近全局最优
  - 速度越慢：计算量 O(k·|Y|·T′) 成正比上升
- **束宽 k 越小**
  - 速度越快
  - 结果越差，容易陷入局部最优
- 工程常用：k=5~10，平衡效果与速度

---

## 习题3：8.5节文本生成用了哪种搜索？可以改进吗？
**答案**：
- 用的是**贪心搜索**（每步选概率最高词）
- 可以改进：
  1. 换成**束搜索**，提升序列全局最优性
  2. 加入 top-k / top-p 采样，增加多样性
  3. 加入长度归一化，避免过短句子

# 三、高频面试题 + 标准答案
## 1. 什么是束搜索？为什么要用它？
**答案**：
束搜索是 seq2seq 推理时的序列搜索策略，每步保留 top-k 候选序列。
用来解决贪心搜索**容易局部最优**、穷举搜索**计算量太大**的问题，平衡效果与速度。

## 2. 贪心、束、穷举搜索的区别？
**答案**：
- 贪心：每步只留1个，最快但不准
- 束搜索：每步留k个，平衡速度与精度
- 穷举：遍历所有，最准但不可用

## 3. 束宽 k 对模型生成有什么影响？
**答案**：
k 越大生成越通顺、接近全局最优，但速度越慢、显存占用越高。
k 越小速度越快，但更容易生成不通顺的局部最优序列。

## 4. 束搜索需要训练吗？为什么？
**答案**：
不需要。
束搜索只是**序列选择算法**，没有可学习参数，只在推理阶段使用。模型（RNN/LSTM/Transformer）单独训练。

## 5. 束搜索为什么要做长度归一化？
**答案**：
- 概率取 log 后都是负数
- 长序列求和更“负”，模型会倾向生成短句子
- 用 L^α 做归一化，消除长度对分数的影响，公平比较不同长度序列

## 6. 为什么 seq2seq 生成不用维特比算法？
**答案**：
1. 自然语言序列**不满足严格马尔可夫性**，当前概率依赖整个历史
2. 维特比复杂度 O(T·|Y|²)，词表很大时计算量无法承受

## 7. 束搜索和 Teacher Forcing 有什么关系？
**答案**：
- Teacher Forcing 是**训练策略**，用真实标签作为输入
- 束搜索是**推理策略**，用模型生成词作为输入
二者完全独立，分别用在训练和推理阶段。

## 8. Beam Search 可能出现什么问题？
**答案**：
- 生成结果**重复、单调**
- 多样性不足，倾向高频短语
- 解决：加大温度系数、用 top-k/top-p 采样、结合多样性惩罚
