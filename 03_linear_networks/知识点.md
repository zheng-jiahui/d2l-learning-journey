# 3.1 线性回归·最完整版基础知识（只讲核心、不绕弯、面试全覆盖）
---
# 一、线性回归是什么？
- 用来预测**连续数值**（房价、温度、销量等）
- 假设输入和输出是**线性关系**
- 模型结构最简单、最基础，是所有深度学习的起点

---

# 二、线性回归的 4 个核心要素
## 1. 模型公式（最基础）
### 单个样本：
$$
\hat{y} = w_1 x_1 + w_2 x_2 + ... + w_d x_d + b
$$
向量化写法：
$$
\hat{y} = \mathbf{w}^\top \mathbf{x} + b
$$

### 批量样本（矩阵形式）：
$$
\hat{\mathbf{y}} = X \mathbf{w} + b
$$
- $X$：数据矩阵（n 个样本，d 个特征）
- $\mathbf{w}$：权重（决定每个特征的重要性）
- $b$：偏置（平移量，让模型更灵活）
- $\hat{y}$：预测值

---

# 三、损失函数
线性回归用 **均方误差 MSE**（最经典）

## 单个样本损失：
$$
l^{(i)}(\mathbf{w},b) = \frac{1}{2} \left( \hat{y}^{(i)} - y^{(i)} \right)^2
$$
- 前面乘 1/2 是为了**求导后系数变成 1**，方便计算。

## 整个数据集平均损失：
$$
L(\mathbf{w},b) = \frac{1}{n} \sum_{i=1}^n l^{(i)}
$$

**目标：找到 w 和 b，让 L 最小。**

---

# 四、优化方法：梯度下降（怎么学）
## 1. 梯度是什么？
- 损失函数对参数 **w、b** 求偏导，得到**梯度**
- 梯度方向 = 损失上升最快方向
- **负梯度方向 = 损失下降最快方向**

## 2. 参数更新公式（最重要）
$$
\mathbf{w} \leftarrow \mathbf{w} - \eta \cdot \nabla L(\mathbf{w})
$$
$$
b \leftarrow b - \eta \cdot \nabla L(b)
$$

- $\eta$：学习率（步长）
- 每一步都往**让损失变小**的方向走一点点

## 3. 小批量随机梯度下降（Mini-batch SGD）
实际训练用这个：
1. 随机抽一小批数据
2. 算这批数据的平均梯度
3. 更新参数
4. 重复直到收敛

---

# 五、线性回归的解析解（数学最优解）
线性回归简单到可以直接用公式算出最优解：
$$
\mathbf{w}^* = (X^\top X)^{-1} X^\top y
$$
- 优点：一步到位
- 缺点：数据量大、特征多的时候**算不动、求逆巨慢**
- 深度学习几乎不用解析解，都用梯度下降

---

# 六、正态分布与均方损失的关系（d2l 重点）
- 假设数据中的噪声服从 **正态分布（高斯分布）**
- 在这个假设下：
  **最小化均方损失 = 最大化似然估计**
所以用 MSE 是有概率依据的，不是随便选的。

---

# 七、线性回归 = 单层神经网络
- 输入层：特征
- 输出层：1 个神经元（预测值）
- 无隐藏层、无激活函数
- 也叫 **全连接层 / 稠密层**

---

# 八、线性回归完整训练流程
1. 初始化 w、b（随机）
2. 前向传播：算预测值 $\hat{y}$
3. 计算损失 MSE
4. 反向传播：算梯度
5. 梯度下降：更新 w、b
6. 重复 2~5 直到损失不再下降

---

# 九、必须记住的 5 个基础定义
1. **特征（x）**：输入信息
2. **标签（y）**：真实值
3. **权重（w）**：特征的重要程度
4. **偏置（b）**：基准值
5. **损失（loss）**：预测错多少
6. **梯度（grad）**：往哪走能变小
7. **学习率（lr）**：每一步走多远

---

# 十、线性回归基础版面试题（带答案）
## 1. 线性回归的模型公式是什么？
$\hat{y} = w^\top x + b$

## 2. 为什么线性回归要用均方误差 MSE？
因为假设噪声服从正态分布，最小化 MSE 等价于最大化似然概率。

## 3. 梯度下降是在做什么？
沿着**负梯度**方向更新参数，让损失不断变小。

## 4. 学习率太大/太小会怎样？
- 太大：不收敛、震荡、发散
- 太小：收敛极慢

## 5. 为什么深度学习不用解析解？
特征多、数据大时，矩阵求逆**计算量爆炸，无法运行**。

## 6. 什么是小批量梯度下降？
每次随机取一小批数据计算梯度并更新，比全量快、比单样本稳。



# 3.2 线性回归·最完整版基础知识（只讲核心、不绕弯、面试全覆盖）
# 一、先讲两个函数的核心思想（对应原文）
## 1. data_iter 函数：批量读取数据（思想）

```
def data_iter(bacth_size, features, labels):
    num_examples = len(features)
    indices = list(range(num_examples))
    # 这些样本是随机读取的，没有特定的顺序
    random.shuffle(indices)
    for i in range(0, num_examples, bacth_size):
        batch_indices = torch.tensor(
            indices[i: min(i+bacth_size, num_examples)]
        )
        yield features[batch_indices], labels[batch_indices]
```

**作用：把大数据切成一小批一小批来训练**
- 数据集太大，不能一次性全部丢进模型
- 每次随机打乱顺序，抽 `batch_size` 个样本
- 一批一批喂给模型，计算梯度、更新参数
- 遍历完所有批次叫 **1 个 epoch**

**原文代码思想：**
```python
def data_iter(batch_size, features, labels):
    1. 把所有样本编号打乱（random.shuffle）
    2. 按 batch_size 切成一段一段
    3. 每次返回一小批 X 和 y
```
**一句话：分批喂数据，不一次性吃完。**

---

## 2. sgd 函数：小批量随机梯度下降（思想）
**作用：用一小批数据的梯度，更新一遍参数**
公式：
```
param = param - lr * param.grad / batch_size
```
代码：
```
def sgd(params, lr, batch_size):  #@save
    """小批量随机梯度下降"""
    with torch.no_grad():
        for param in params:
            param -= lr * param.grad / batch_size
            param.grad.zero_()
```
- `param.grad`：这一批数据算出来的梯度
- `/ batch_size`：把梯度平均到每个样本
- `lr`：学习率，控制每一步走多远

**一句话：沿着梯度反方向走一小步，让损失变小。**

# 3.4 Softmax 回归（完整知识点+面试题+标准答案）
---

## 一、Softmax 回归是什么
- 解决**多分类问题**的线性模型
- 输出每个类别的**概率**（和为1，均≥0）
- 结构：线性层 + Softmax 归一化

---

## 二、独热编码 One-Hot
类别标签用向量表示，只有对应位置为1：
- 类别0 → [1,0,0]
- 类别1 → [0,1,0]
- 类别2 → [0,0,1]

目的：让类别无大小顺序，适合概率计算。

---

## 三、模型计算过程
1. **线性输出**
\[
\mathbf o = \mathbf W \mathbf x + \mathbf b
\]
2. **Softmax 归一化**
\[
\hat y_j = \frac{\exp(o_j)}{\sum_{k} \exp(o_k)}
\]
- 把任意实数 → 概率分布
- 不改变大小顺序，可导

---

## 四、损失函数：交叉熵 Cross-Entropy
\[
l(\mathbf y, \hat{\mathbf y}) = -\sum_j y_j \log \hat y_j
\]
因真实标签是独热，简化为：
\[
l = -\log(\hat y_{\text{true}})
\]
思想：让**正确类别的预测概率尽可能大**。

---

## 五、梯度（核心）
\[
\frac{\partial l}{\partial o_j} = \hat y_j - y_j
\]
预测概率 − 真实标签，简洁易算。

---

## 六、与线性回归的区别
- 线性回归：预测连续值，用 MSE
- Softmax 回归：多分类，用 Softmax + 交叉熵

---

# 高频面试题 + 标准答案（必背）

## 1. Softmax 函数的作用是什么？
将模型原始输出转换为**非负且和为1的概率分布**，满足概率公理，同时可导，支持反向传播。

## 2. 为什么多分类用交叉熵损失，不用 MSE？
- MSE 对错误分类惩罚不足，梯度小，训练慢
- 交叉熵来自**极大似然估计**，更适合概率输出
- 梯度形式简单、训练更稳定

## 3. 为什么 Softmax 要先 exp？
- exp 保证输出**非负**
- 单调递增，不改变类别大小顺序
- 便于构建概率分布并求导

## 4. Softmax 回归的梯度是什么？
对线性输出 \(o_j\) 的梯度为：
\[
\hat y_j - y_j
\]
即预测概率减去真实标签。

## 5. Softmax 回归和逻辑回归的关系？
- 逻辑回归用于**二分类**
- Softmax 回归用于**多分类**
- 二分类时 Softmax 等价于 Sigmoid + 二元交叉熵

## 6. 为什么训练时常常只取 argmax，不用 Softmax？
因为 Softmax**不改变大小顺序**，预测类别只看最大值即可，节省计算。

## 7. Softmax 会不会出现数值溢出？怎么解决？
会，输入过大时 exp 会爆炸。
解决方法：**每行减去最大值**再计算 Softmax，不改变结果但稳定数值。

### 1. Fashion-MNIST 和 MNIST 有什么区别？
- MNIST：手写数字（0-9），过于简单，容易过拟合

- 将 PIL 图像 / numpy 数组 **转为 PyTorch Tensor**

- 每个 epoch 随机打乱数据顺序

- 训练：需要随机，保证梯度无偏

- 1：通道数（灰度图单通道）

- 太小：梯度噪声大，训练不稳定，占用显存少


# 3.6 Softmax回归从零实现（核心总结+疑问一次讲清）

## 代码分解

```python
return - torch.log(y_hat[range(len(y_hat)), y])
```

### 1. `y_hat` 的形状
假设：
- `y_hat` 是模型的预测输出，形状为 `(batch_size, num_classes)`
- 每个样本的预测值经过 softmax 后，表示属于各个类别的概率

例如：
```python
y_hat = torch.tensor([[0.1, 0.2, 0.7],
                      [0.8, 0.1, 0.1]])
# batch_size=2, num_classes=3
```

### 2. `y` 的形状
`y` 是真实标签，形状为 `(batch_size,)`，每个元素是类别索引

例如：
```python
y = torch.tensor([2, 0])  # 第一个样本真实类别是2，第二个样本真实类别是0
```

### 3. `range(len(y_hat))`
- `len(y_hat)` = batch_size = 2
- `range(2)` 生成 `[0, 1]`

### 4. 索引操作 `y_hat[range(len(y_hat)), y]`
这使用了高级索引，选取每个样本对应真实类别的预测概率：

```python
# y_hat[range(2), y] 等价于：
# 样本0: y_hat[0, y[0]] = y_hat[0, 2] = 0.7
# 样本1: y_hat[1, y[1]] = y_hat[1, 0] = 0.8
# 结果: tensor([0.7, 0.8])
```

### 5. `- torch.log(...)`
计算负对数似然：
```python
# -torch.log([0.7, 0.8]) = -[-0.3567, -0.2231] = [0.3567, 0.2231]
```

## 完整示例

```python
import torch

# 假设有3个类别，batch_size=2
y_hat = torch.tensor([[0.1, 0.2, 0.7],   # 样本0的预测概率
                      [0.8, 0.1, 0.1]])  # 样本1的预测概率

y = torch.tensor([2, 0])  # 真实标签

# 计算交叉熵损失
loss = - torch.log(y_hat[range(len(y_hat)), y])
print(loss)  # tensor([0.3567, 0.2231])

# 取平均得到最终损失
final_loss = loss.mean()
print(final_loss)  # tensor(0.2899)
```

## 数学原理

这就是交叉熵损失函数：
$$\text{loss} = -\sum_{i=1}^{n} \log(\hat{y}_i[y_i])$$

其中：
- `n` 是 batch_size
- `\hat{y}_i[y_i]` 是第 `i` 个样本预测为真实类别 `y_i` 的概率

## 等价写法

这种写法等价于：
```python
# 方法1：使用 gather
loss = - torch.log(torch.gather(y_hat, 1, y.unsqueeze(1))).squeeze()

# 方法2：使用 cross_entropy 函数
loss = torch.nn.functional.cross_entropy(y_hat, y, reduction='none')
```

这种索引方式比循环更高效，是 PyTorch 中实现交叉熵损失的常见写法。



## 1. 为什么一会儿 sum 一会儿 mean？

这取决于损失函数 `l` 的定义方式：

### 情况1：使用自定义损失函数（else分支）
```python
l = loss(y, y_hat)  # 这里 loss 是 squared_loss
```
如果你的 `loss` 返回的是**每个样本的损失向量**（形状为 `(batch_size,)`）：
```python
def squared_loss(y_hat, y):
    return (y_hat - y.reshape(y_hat.shape)) ** 2 / 2  # 返回向量，不是标量
```

那么：
- **反向传播前必须求和**：`l.sum().backward()` 
- 原因：PyTorch 的 `.backward()` 只能对标量调用，不能对向量调用

### 情况2：使用 PyTorch 内置损失函数（if分支）
```python
l = loss(y, y_hat)  # 这里 loss 是 nn.MSELoss() 或 nn.CrossEntropyLoss()
```
PyTorch 内置损失函数默认返回**标量**（已经对所有样本求平均）：
```python
criterion = nn.MSELoss(reduction='mean')  # 默认就是 'mean'
l = criterion(y_hat, y)  # l 已经是标量
```

那么：
- **可以直接反向传播**：`l.backward()` 
- 不需要再 `.sum()` 或 `.mean()`

## 2. updater(X.shape[0]) 是什么意思？

这是在使用**自定义优化器**（如你之前代码中的 `sgd` 函数）时的参数传递。

### 查看你的 sgd 函数定义：
```python
def sgd(params, lr, batch_size):
    with torch.no_grad():
        for param in params:
            param -= lr * param.grad / batch_size  # 需要 batch_size
            param.grad.zero_()
```

### 调用方式：
```python
# 在 train_epoch_ch3 中
else:
    # 使用定制的优化器和损失函数
    l.sum().backward()
    updater(X.shape[0])  # updater 是 sgd 函数，传入 batch_size
```

- `updater` 是传入的函数，比如 `sgd`
- `X.shape[0]` 是当前批次的样本数（batch_size）
- 调用 `updater(batch_size)` 相当于 `sgd([w, b], lr, batch_size)`

## 总结

| 组件 | 自定义方式 | PyTorch 内置方式 |
|------|-----------|-----------------|
| 损失函数 | 返回向量 | 返回标量（已平均） |
| 反向传播 | `l.sum().backward()` | `l.backward()` |
| 优化器 | 手动除以 batch_size | 自动处理 |
| 调用方式 | `updater(batch_size)` | `optimizer.step()` |

这样设计是为了保持灵活性，既支持自定义实现（教学目的），也支持 PyTorch 内置功能（实际应用）。


# 三、高频面试题（必背）
1. **为什么要展平图像？**
   因为Softmax回归是**线性模型**，只能处理向量，不能处理2D图像。

2. **Softmax作用是什么？**
   把任意实数输出变成 **非负、和为1的概率分布**。

3. **为什么用交叉熵，不用MSE？**
   交叉熵来自**极大似然估计**，更适合分类概率，梯度更大、训练更稳定。

4. **sum和mean损失区别？**
   只是梯度缩放不同，**只要配套除以batch_size就等价**。

# 3.7 softmax回归的简洁实现
---
## 代码分解

```python
net = nn.Sequential(
    nn.Flatten(), 
    nn.Linear(784, 10)
)
```

## 1. `nn.Sequential`

这是一个**容器**，按顺序包装多个网络层：
- 数据会依次通过每一层
- 前一层的输出自动作为后一层的输入

```python
# 等价于：
def forward(x):
    x = nn.Flatten()(x)      # 第一步：展平
    x = nn.Linear(784, 10)(x) # 第二步：全连接
    return x
```

## 2. `nn.Flatten()` - 展平层

**作用**：将多维输入展平为一维

### MNIST 数据格式
MNIST 手写数字图片：
- 原始图像：28×28 像素
- 形状：`(batch_size, 1, 28, 28)` 
  - batch_size：一次输入多少张图
  - 1：灰度通道（RGB是3）
  - 28：高度
  - 28：宽度


**为什么要展平？**
- `nn.Linear` 层（全连接层）期望输入是二维的：`(batch_size, features)`
- 每个样本的特征必须是向量形式

## 3. `nn.Linear(784, 10)` - 全连接层

**语法**：`nn.Linear(in_features, out_features)`

- `in_features=784`：输入特征数（每个样本有 784 个像素值）
- `out_features=10`：输出特征数（10 个数字类别 0-9）

---

## 损失s的reduction 参数的三个选项

### 1. `reduction='mean'`（默认值）
返回所有样本损失的**平均值**（标量）

### 2. `reduction='sum'`
返回所有样本损失的**总和**（标量）

### 3. `reduction='none'`
返回**每个样本的损失**（向量），不进行汇总

## 为什么需要 reduction='none'？

### 1. **自定义聚合方式**
有时需要加权平均或特殊处理：

```python
# 给不同样本不同的权重
criterion = nn.CrossEntropyLoss(reduction='none')
loss_per_sample = criterion(y_hat, y)

# 自定义权重（例如，某些样本更重要）
sample_weights = torch.tensor([1.0, 2.0])  # 第二个样本权重更高
weighted_loss = (loss_per_sample * sample_weights).mean()
```

## 总结

| reduction 选项 | 输出形状 | 用途 |
|---------------|---------|------|
| `'mean'` | 标量 | 标准训练，最常用 |
| `'sum'` | 标量 | 需要总损失时 |
| `'none'` | 向量 (batch_size,) | 需要逐样本分析、自定义聚合、高级训练技巧 |

**记忆要点**：
- `'mean'` 和 `'sum'` 会自动聚合成标量
- `'none'` 保持向量形式，让你有更多控制权
- 使用 `'none'` 后，记得手动调用 `.mean()` 或 `.sum()` 再 `.backward()`

# 三、面试常见问题（必背）
### 1. 为什么CrossEntropyLoss不用手动加softmax？
因为 **CrossEntropyLoss = log_softmax + nll_loss**，
框架内部已经集成了softmax，只需要输入模型原始输出（logits）。

### 2. 为什么要Flatten()？
把 [batch, 1, 28, 28] 的图像展平成 [batch, 784]，
才能输入全连接层 `nn.Linear`。


### 4. 为什么训练时输出不用做softmax？
因为 **CrossEntropyLoss 会自动处理**，
手动做反而会导致：
- 数值不稳定
- 损失计算错误
- 收敛变慢

### 5. 什么叫 from_logits？
表示输入是**未经过softmax的原始输出**，
让损失函数内部做归一化，保证数值安全。

# 四、课后题答案（官方章节练习题）

## 练习1：调整 batch_size、epochs、lr 会发生什么？
- **batch_size 太小**：训练震荡、噪声大
- **batch_size 太大**：梯度更准，但速度变慢
- **lr 太大**：不收敛、来回跳
- **lr 太小**：收敛极慢
- **epochs 适中**：精度上升后平稳
- **epochs 太多**：可能过拟合，测试精度下降

## 练习2：为什么迭代轮数变多，测试精度会下降？怎么解决？
### 原因：
**过拟合（overfitting）**
模型在训练集上学得太细，把噪声也学会了，在测试集上泛化能力变差。

### 解决方法：
1. **早停（early stop）**：精度不再上升就停
2. **权重衰减（L2正则）**
3. **数据增强**
4. **降低模型复杂度**
5. **加dropout**
