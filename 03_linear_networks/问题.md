## 理论题

### 1. 为什么 ReLU 比 Sigmoid 更常用？有什么缺点？

**ReLU 的优势**：
- **缓解梯度消失**：当 x > 0 时导数为 1，梯度能有效反向传播
- **计算简单**：max(0,x)，无指数运算，训练更快
- **稀疏激活**：部分神经元输出为 0，增强模型表达能力

**ReLU 的缺点**：
- **神经元死亡**（Dying ReLU）：当输入为负时梯度为 0，若学习率过大，大量神经元可能永久失活
- **输出无界**：可能导致数值不稳定

**改进版本**：LeakyReLU、PReLU、ELU 等

---

### 2. Dropout 的原理是什么？训练和测试时的行为有何不同？

**原理**：
- 训练时以概率 p 随机丢弃神经元（将其输出置 0），防止神经元间共适应，强制网络学习更鲁棒的特征

**训练 vs 测试**：
| 阶段 | 行为 |
|------|------|
| **训练** | 以概率 p 随机丢弃，保留的神经元输出缩放为 \( \frac{1}{1-p} \)（或测试时缩放） |
| **测试** | 使用全部神经元，不进行丢弃，输出乘以 \( 1-p \)（若训练时未缩放） |

PyTorch 中通过 `model.train()` 和 `model.eval()` 自动切换。

---

### 3. L1 正则化和 L2 正则化在效果和梯度更新上有何区别？

| 特性 | **L1 正则化** | **L2 正则化** |
|------|---------------|---------------|
| **公式** | \( \lambda \sum \|w\|_1 \) | \( \frac{\lambda}{2} \sum w^2 \) |
| **梯度更新** | \( w \leftarrow w - \eta (\nabla L + \lambda \cdot \text{sign}(w)) \) | \( w \leftarrow w - \eta (\nabla L + \lambda w) \) |
| **效果** | 产生**稀疏解**，部分权重为 0，可用于特征选择 | 权重趋向于**均匀变小**但不为 0 |
| **几何解释** | 菱形约束，易在坐标轴处相交 | 圆形约束，平滑收缩 |
| **鲁棒性** | 对异常值更鲁棒 | 平滑优化，更稳定 |

---

### 4. 什么是梯度裁剪（gradient clipping），在什么情况下用？

**定义**：当梯度的范数超过阈值时，将梯度按比例缩放到阈值范围内。

**公式**：
\[
\text{if } \|g\| > \text{max\_norm}: \quad g \leftarrow \text{max\_norm} \cdot \frac{g}{\|g\|}
\]

**使用场景**：
1. **梯度爆炸**：尤其在 RNN/LSTM 等循环网络中常见
2. **深层网络训练**：防止梯度过大导致训练不稳定
3. **对抗训练**：控制扰动幅度

PyTorch 实现：
```python
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

## 代码题

### 1. 在 PyTorch 中如何添加 L2 正则化（weight decay）？

**方法一：优化器参数**
```python
optimizer = torch.optim.SGD(model.parameters(), lr=0.01, weight_decay=1e-4)
```
`weight_decay` 即为 L2 正则化系数 λ。

**方法二：手动添加（不推荐）**
```python
l2_lambda = 1e-4
l2_reg = sum(p.norm(2)**2 for p in model.parameters())
loss = criterion(output, target) + l2_lambda * l2_reg
```

### 2. 写出一个带 dropout 的两层 MLP 的 nn.Module 定义

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class TwoLayerMLP(nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim, dropout_rate=0.5):
        super().__init__()
        self.fc1 = nn.Linear(input_dim, hidden_dim)
        self.fc2 = nn.Linear(hidden_dim, output_dim)
        self.dropout = nn.Dropout(dropout_rate)
    
    def forward(self, x):
        x = self.fc1(x)
        x = F.relu(x)
        x = self.dropout(x)      # 只在训练时生效
        x = self.fc2(x)
        return x

# 使用
model = TwoLayerMLP(784, 256, 10, dropout_rate=0.5)
```

### 3. 如何在 PyTorch 中实现自定义的权重初始化（如 Xavier 初始化）

**方法一：在 `__init__` 中逐层初始化**
```python
def _init_weights(self):
    for m in self.modules():
        if isinstance(m, nn.Linear):
            nn.init.xavier_uniform_(m.weight)
            if m.bias is not None:
                nn.init.zeros_(m.bias)

class MyModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(784, 256)
        self.fc2 = nn.Linear(256, 10)
        self._init_weights()
    
    def _init_weights(self):
        nn.init.xavier_uniform_(self.fc1.weight)
        nn.init.zeros_(self.fc1.bias)
        nn.init.xavier_uniform_(self.fc2.weight)
        nn.init.zeros_(self.fc2.bias)
```

**方法二：使用 `apply` 递归初始化**
```python
def xavier_init(m):
    if isinstance(m, nn.Linear):
        nn.init.xavier_uniform_(m.weight)
        if m.bias is not None:
            nn.init.zeros_(m.bias)

model = MyModel()
model.apply(xavier_init)
```

## 场景题

### 1. 训练时发现训练 loss 下降但验证 loss 上升，你认为是过拟合，会采取什么措施？

**立即措施**：
1. **早停（Early Stopping）**：保存验证集最佳模型
2. **增加 Dropout**：提高 dropout 比例
3. **增加 L2 正则化**：增大 weight_decay
4. **降低模型复杂度**：减少层数或隐藏单元数

**中期调整**：
5. **数据增强**：增加训练数据多样性
6. **集成学习**：训练多个模型取平均
7. **学习率衰减**：使用学习率调度器

**根本解决**：
8. **增加训练数据**：收集更多标注数据
9. **简化特征**：特征选择或降维

---

### 2. 如果你的 MLP 参数量远大于样本量，怎么避免过拟合？

**核心策略**（按优先级排序）：

**1. 强正则化**
- **Dropout**：设置较高的 dropout 率（0.5~0.8）
- **L2 正则化**：增大 weight_decay（如 1e-3 ~ 1e-2）
- **L1 正则化**：产生稀疏权重，自动特征选择

**2. 模型压缩**
- **减少隐藏层数量**：用 1-2 层代替多层
- **减少每层神经元数**
- 使用**瓶颈结构**（bottleneck）

**3. 数据层面**
- **数据增强**：即使对 MLP，也可添加噪声
- **伪标签**：利用未标注数据

**4. 训练技巧**
- **早停**：验证集 loss 不再下降时停止
- **降低学习率**：小学习率 + 长训练
- **小批量大小**：增加梯度随机性

**5. 集成方法**
- **Bagging**：训练多个子模型取平均
- **交叉验证**：确保模型泛化能力

**示例配置**：
```python
model = TwoLayerMLP(input_dim=784, hidden_dim=64, output_dim=10, dropout_rate=0.7)
optimizer = torch.optim.Adam(model.parameters(), lr=0.001, weight_decay=1e-2)
# 使用早停回调
```
