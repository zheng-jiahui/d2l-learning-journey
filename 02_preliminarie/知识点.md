# 一、2.1 ndarray 知识点完整总结
本节目标：掌握张量（Tensor）的**创建、形状、运算、索引、内存、数据转换**，是后面所有模型代码的基础。

## 1. 张量（Tensor）基本概念
- 张量就是**多维数组**，对标 NumPy 的 ndarray。
- 0维：标量；1维：向量；2维：矩阵；3维及以上：张量。
- 深度学习里所有数据（图片、文本、特征）都用张量存储。

## 2. 基本操作
1. **创建张量**
   - `torch.arange(n)`：生成 0~n-1 的序列
   - `zeros((m,n))`、`ones((m,n))`：全0、全1
   - `randn(m,n)`：正态分布随机数

2. **形状操作**
   - `shape`：查看形状
   - `numel()`：总元素个数
   - `reshape(h, w)`：改变形状，不改变数据
   - `reshape(-1, c)`：`-1` 让框架自动计算维度

3. **索引与切片**
   - `x[0]`、`x[1:3]`、`x[:, 1]` 读取
   - 可直接赋值修改：`x[0,0] = 5`

4. **按元素运算**
   - `+ - * / **` 都是逐元素运算
   - `exp`、`log` 等数学函数

5. **张量拼接**
   - `torch.cat([X, Y], dim=0)`：按行堆叠（上下拼）
   - `torch.cat([X, Y], dim=1)`：按列堆叠（左右拼）

6. **逻辑运算**
   - `X == Y` 会逐元素比较，返回布尔张量

7. **求和**
   - `x.sum()`：所有元素求和

8. **内存节省（原地操作）**
   - 普通操作 `Y = X + Y` 会开辟新内存
   - `Y[:] = X + Y` 或 `Y += X` 原地操作，节省内存

9. **数据类型转换**
   - Tensor ↔ NumPy 数组互转
   - 标量张量 → Python 标量：`.item()`

# 二、广播机制（Broadcast Mechanism）重点总结
这一节里明确讲了：**即使形状不同，张量也可以按元素运算**，机制叫广播。

## 1. 广播发生的条件
两个张量进行逐元素运算时，如果：
1. 从**末尾维度**向前对比；
2. 维度相等，或其中一个维度为 1；
就会触发广播。

## 2. 广播的过程
- 把维度为 1 的那一维**复制扩展**，直到两个张量形状一致
- 然后再逐元素运算
- **不会真的复制数据，只是逻辑上扩展，节省内存**

## 3. 书中典型例子
```
a = torch.arange(3).reshape((3,1))
b = torch.arange(2).reshape((1,2))
a + b
```
- a 形状 (3,1)
- b 形状 (1,2)
广播后都变成 (3,2)，再相加。

## 4. 一句话记住
**维度从后往前看，要么相等，要么有一个是1，就能广播。**

# 三、NumPy ndarray 和 PyTorch Tensor 的区别（面试必背）
## 1. 硬件支持
- NumPy：**仅支持 CPU**
- Tensor：支持 **CPU / GPU / TPU**，可 `.to('cuda')` 移到显卡

## 2. 自动求导
- NumPy：**无自动微分**，梯度要手动推导
- Tensor：内置 Autograd，`requires_grad=True` + `backward()` 自动算梯度

## 3. 用途定位
- NumPy：通用科学计算
- Tensor：专为深度学习设计，配合网络层、优化器、数据集等

## 4. 设备与内存
- NumPy：始终在 CPU 内存
- Tensor：可在设备间迁移，支持分布式训练

## 5. API 相似但不完全一致
- 语法高度相似，容易上手
- 但函数名、属性略有不同：
  - NumPy：`.size`
  - PyTorch：`.numel()`

## 4. 降维与求和（本章重点 + 面试必考）
### 4.1 求和函数
- `x.sum()`：对**所有元素**求和，结果是标量（完全降维）。
- `x.sum(axis=d)`：沿**第 d 个轴**求和，该轴被消去。
- 举个栗子，沿行求和，就是固定某一列，一行一行下来求和

# 🔥 超详细：按行求和、按列求和（2维情况）
以形状为 **(行数, 列数) = (m, n)** 的矩阵为例：

## （1）沿 axis=0 求和 → **按列求和**
- `axis=0` 是**行方向**
- 沿着行往下扫，把同一列的所有行加在一起
- 结果形状：**(n,)**，列数保留，行数消失

```
A = [[a11,a12,a13],
     [a21,a22,a23],
     [a31,a32,a33]]    # shape (3,3)

A.sum(axis=0)
= [a11+a21+a31, a12+a22+a32, a13+a23+a33]
# shape (3,)
```

一句话：
**axis=0 求和 = 按列求和 = 每列变成一个数 = 行数被消掉**

---

## （2）沿 axis=1 求和 → **按行求和**
- `axis=1` 是**列方向**
- 沿着列往右扫，把同一行的所有列加在一起
- 结果形状：**(m,)**，行数保留，列数消失

```
A.sum(axis=1)
= [a11+a12+a13, a21+a22+a23, a31+a32+a33]
# shape (3,)
```

一句话：
**axis=1 求和 = 按行求和 = 每行变成一个数 = 列数被消掉**

---

## （3）keepdims=True（保持维度）
默认求和会**消去轴**，形状从 (m,n) → (n,) 或 (m,)。

加 `keepdims=True` 后：
- 轴不被删除，只是长度变成 1
- 形状从 (m,n) → **(1,n)** 或 **(m,1)**
- 方便后续**广播运算**

例子：
```
A.sum(axis=0, keepdims=True)  # shape (1,3)
A.sum(axis=1, keepdims=True)  # shape (3,1)
```

# 二、完整知识点

## 1. 导数（Derivative）
- 定义：函数在某点的**瞬时变化率**
- 几何意义：**切线斜率**
- 意义：告诉我们输入微小变化时，输出怎么变

## 2. 偏导数（Partial Derivative）
- 函数有**多个自变量**时，对**其中一个变量求导**，其他视为常数
- 深度学习中，损失函数有很多参数（w₁,w₂,…），每个参数都要算偏导

## 3. 梯度（Gradient）
- 定义：**函数对所有变量的偏导数组成的向量**
- 记法：∇f
- 意义：
  - 梯度方向：函数**上升最快**的方向
  - 负梯度方向：函数**下降最快**的方向
- 这就是**梯度下降**的来源：往负梯度走，让损失变小

## 4. 链式法则（Chain Rule）【本节最重要】
复合函数求导规则：
\[
\frac{dy}{dx} = \frac{dy}{du} \cdot \frac{du}{dx}
\]

多层网络时：
损失 → 最后一层 → … → 第一层
求导就是**一层一层连乘**
→ 这就是**反向传播（Backpropagation）**的数学本质

## 5. 梯度在深度学习中的意义
- 损失函数 L(θ) 关于参数 θ 的梯度：∇L(θ)
- 参数更新公式：
\[
θ = θ - η \cdot ∇L(θ)
\]
- η：学习率
- 朝着**损失下降**的方向走

## 6. 高维情况
- 标量函数对向量求导 → 得到**梯度向量**
- 向量对向量求导 → 得到**雅可比矩阵**
- 深度学习里主要用**标量损失对参数向量求梯度**

# 三、本节高频面试题 + 标准答案

## 1. 梯度是什么？和导数、偏导数有什么区别？
- 导数：对**单个变量**的变化率
- 偏导数：多变量函数对**某一个变量**的导数
- 梯度：**所有偏导数组成的向量**，代表函数变化最剧烈的方向

## 2. 梯度的方向代表什么？
- 梯度方向：函数**上升最快**
- 负梯度方向：函数**下降最快**
所以梯度下降要沿**负梯度**更新参数。

## 3. 什么是链式法则？为什么深度学习离不开它？
- 链式法则是复合函数求导的规则：逐层求导再相乘。
- 深度网络是多层复合函数，反向传播就是用链式法则**从后往前算梯度**。

## 4. 梯度下降中为什么要沿负梯度方向更新？
因为负梯度是**损失函数下降最快**的方向，能最快减小误差。

## 5. 学习率 η 太大或太小会怎样？
- 太大：不收敛、震荡、甚至发散
- 太小：收敛极慢，容易卡在局部极小

## 6. 梯度和反向传播的关系是什么？
反向传播就是**利用链式法则高效计算所有参数的梯度**，没有梯度就无法更新网络。

# 2.5 自动求导 完整总结

# 一、本节核心内容
自动求导（Automatic Differentiation）是深度学习框架的**核心能力**，让我们不用手动推导反向传播公式，框架自动计算梯度。

## 1. 核心机制：计算图
- 框架会把运算记录成一张**计算图**
- 前向传播：构建图、计算输出
- 反向传播：从输出往回走，用**链式法则**自动算所有参数的梯度

## 2. 梯度缓存与累积
- PyTorch 中梯度默认**累加（累积）**，不会自动清空
- 所以每次迭代前要用 `optimizer.zero_grad()` 或 `w.grad.zero_()`

## 3. 分离计算：detach()
- 切断计算图，让某些变量**不参与求导**
- 用途：冻结部分网络、计算指标时不影响梯度

## 4. 控制流的自动求导
- Python 的 `for` / `if` 控制流，PyTorch 依然能正确求导
- 每次运行都会动态构建计算图，这是 PyTorch 动态图的优势

## 5. 梯度的本质
- 标量对向量求导 → 得到梯度向量
- 只有**标量损失**才能直接 `.backward()`
- 向量求导需要指定 `gradient` 参数

---

# 二、关键 API 与含义（必须记）
1. **`requires_grad=True`**
   - 告诉框架：需要对这个张量**追踪计算、计算梯度**
2. **`.backward()`**
   - 启动反向传播，自动计算所有叶子节点梯度
3. **`.grad`**
   - 存放计算好的梯度
4. **`.detach()`**
   - 截断梯度流，返回不需要求导的新张量
5. **`with torch.no_grad():`**
   - 上下文管理器，**禁用梯度计算**
   - 用于推理、评估，节省显存、加速

# 三、本节高频面试题 + 标准答案

## 1. 什么是自动求导？为什么深度学习离不开它？
自动求导是框架自动根据计算图和链式法则，计算出所有参数梯度的机制。
深度网络层数多、函数复杂，手动求导几乎不可能，自动求导让训练变得可行。

## 2. `requires_grad=True` 作用是什么？
标记该张量**需要被追踪梯度**，框架会记录它参与的所有运算，以便后续反向传播计算梯度。

## 3. 为什么每次反向传播前要清空梯度？
PyTorch 梯度会**累积（累加）**，不清空会导致梯度错误叠加，参数更新异常。

## 4. `detach()` 和 `torch.no_grad()` 有什么区别？
- `detach()`：返回一个**不再参与计算图**的张量，常用于切断部分支路梯度
- `torch.no_grad()`：是一个**上下文环境**，在其中所有运算都不构建计算图、不存梯度
- 前者针对**单个张量**，后者针对**一段代码**

## 5. 非标量张量如何调用 `backward()`？
向量/矩阵调用 `.backward()` 时，必须传入**梯度参数**，用于加权求和转为标量。
实际训练中我们几乎只用**标量损失**调用 backward。

## 6. 叶子张量和非叶子张量的区别？
- 叶子张量：用户手动创建、`requires_grad=True` 的张量（如权重 w）
- 非叶子张量：运算生成的中间变量
- 只有**叶子节点**的梯度会被保留，中间变量梯度默认被释放

## 7. 为什么推理时要用 `torch.no_grad()`？
- 不构建计算图，**节省大量显存**
- 不计算梯度，**前向速度更快**
- 避免无意修改参数

# 一、什么是分离计算？detach() 到底在干嘛？

## 一句话核心

**detach() = 把一个张量从计算图上“剪下来”，让它不再参与梯度计算。**

更细一点：
- 一个 tensor 如果是通过计算得到的（比如 `y = x * 2`），它就**挂在计算图上**
- 梯度会从 loss 一路传到 y，再传到 x
- 一旦你调用 **y.detach()**
  - 会返回一个**一模一样数值**的 tensor
  - 但是：**不再连接计算图、不再记录梯度、requires_grad=False**
  - 梯度流到这里就**断了**，再也传不回去

## 作用是什么？（面试常问）
- 不想让某些部分参与更新（冻结层）
- 计算指标、统计量时不想影响梯度
- 节省显存，不让计算图无限变大

## 举个超级简单例子
```python
x = torch.tensor(2.0, requires_grad=True)
y = x * 2
z = y ** 2

z.backward()
print(x.grad)   # 结果：16 → 梯度正常传到 x
```

现在加 detach：
```python
x = torch.tensor(2.0, requires_grad=True)
y = x * 2
y = y.detach()    # 👈 从计算图剪断，不仅x得不到，y自己也不会更新
z = y ** 2

z.backward()
print(x.grad)   # 结果：None
```
因为梯度到 y 就断了，**传不到 x**。

# 二、为什么对 sum 求导时，传入的外部梯度是 1？
这是 d2l 2.5 里最容易懵的点：

```python
x = torch.arange(4.0, requires_grad=True)
y = 2 * x
y.sum().backward()
```

为什么等价于：
```python
y.sum().backward(gradient=torch.ones_like(y)) #gradient = 上游传过来的梯度（dL/dy）
```

## 核心原理
1. **`.backward()` 只能从标量开始反向**
2. `y.sum()` 本身已经是**标量**
3. 对标量反向传播，框架默认外部“上游梯度”就是 **1**

### 为什么是 1？
反向传播公式是：
\[
\frac{\partial L}{\partial x} = \frac{\partial L}{\partial z} \cdot \frac{\partial z}{\partial x}
\]

这里：
- \(z = y.sum()\) 是标量
- 我们就是对 z 反向，所以 \(\frac{\partial L}{\partial z}=1\)

所以框架默认填 **1**。

### 几何理解
sum 就是把所有输出加在一起变成一个数，
对这个数求导，相当于**每个位置的权重都是 1**。

# 三、你要的 2 道附加题（面试级）
## 问题1：`y.sum().backward()` 和 `y.backward(torch.ones_like(y))` 为什么结果一样？
### 答案
- `y.sum()` 先把 y 求和变成标量，再反向，外部梯度=1
- 直接 `y.backward(gradient=1)` 相当于：
  把每个分量的上游梯度都设为 1，再分别传回 x
数学上完全等价：
\[
\frac{\partial}{\partial x}\left(\sum_i y_i\right)
= \sum_i \frac{\partial y_i}{\partial x}
\]

## 问题2：为什么非标量张量 backward 必须传入 gradient 参数？
### 答案
- 非标量（向量/矩阵）不是一个值，框架不知道**每个分量的权重**
- gradient 就是用来表示：
  \(\frac{\partial L}{\partial y_1}, \frac{\partial L}{\partial y_2}, \dots\)
- 不传入，框架不知道怎么加权合成最终梯度
- 深度学习里永远用**标量 loss**反向，所以平时不用手动传 gradient

# 四、本节终极 3 问 + 标准答案（必背）
## 1. detach() 的作用是什么？
从计算图中分离张量，**切断梯度回流**，使其不再参与求导，返回一个无梯度的新张量。

## 2. 为什么对标量 loss 反向不需要传 gradient？
因为对标量反向时，框架默认上游梯度为 **1**，足够完成链式法则。

## 3. detach() 和 torch.no_grad() 区别？
- detach()：针对**单个张量**，剪断梯度流
- torch.no_grad()：针对**一段代码**，全局不构建计算图
- 前者细粒度，后者粗粒度

对，**一句话说死：**
一旦调用了 **y.detach()**，不仅梯度**传不到前面的变量**，连 **y 自己的梯度也不会被计算、不会被保存**。。
---

# 1. y.detach() 之后，y 的梯度还会更新吗？
**不会。完全不会。**

因为：
- detach() 会返回一个新张量
- 这个新张量的 **requires_grad = False**
- 它**不再属于计算图**
- 所以反向传播时，框架根本不会为它计算梯度
- **y.grad 永远是 None**

梯度从它这里**直接断掉**，既不往前传，也不留在自己身上。

---

# 2. 最直观的例子（一看就懂）
```python
x = torch.tensor(1.0, requires_grad=True)
y = x * 3   # y 依赖x
y = y.detach()  # 分离
z = y + 2
z.backward()

print(x.grad)   # None（梯度断了）
print(y.grad)   # None（自己也没有梯度）
```

---

# 3. 你最关心的结论
**y.detach() = y 不再参与梯度计算
→ y 自己不会有梯度
→ 前面的变量也收不到梯度**

# 4. 我再顺便回答你之前没好意思问的关键点
## 为什么 y.sum().backward() 外部梯度是 1？
因为：
- `backward()` 必须从**标量**开始反向
- `y.sum()` 是一个标量 L
- 对 L 反向时，最顶层梯度默认就是
  \[
  \frac{\partial L}{\partial L} = 1
  \]
所以框架自动给 **1**。

## 问题1：detach() 之后，原来的 y 还在计算图里吗？
不在了。

- **detach()**：只切断某一个张量的梯度
- **torch.no_grad()**：一段代码都不构建计算图
