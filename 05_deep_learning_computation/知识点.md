# 5.1 模型构造（Model Construction）
1. **块（Block）**
   神经网络的基本封装单位，可以是一层、多层、整个网络。
   核心：**管理参数 + 定义前向传播**。

2. **Sequential 容器**
   按顺序堆叠层，自动实现前向传播，适合简单线性网络。
   局限：**只能串行，不能写复杂逻辑**。

3. **自定义块（继承 nn.Module）**
   必须实现两个方法：
   - `__init__`：定义子层和参数
   - `forward`：定义前向计算逻辑
   优点：支持**if/for/循环/复用层**等任意控制流。

4. **块的嵌套**
   块里可以包含块，实现复杂网络（ResNet、GoogLeNet）。

5. **为什么不用普通 list 存层？**
   用 list 存放层，**参数不会被模型自动管理**，导致：
   - 无法用 `to(device)` 移到 GPU
   - 无法用 `parameters()` 收集参数
   - 无法保存加载模型

---

# 5.2 参数管理（Parameters）
1. **参数本质**
   模型可学习的变量：`weight`、`bias`，都是 `nn.Parameter` 类型。

2. **核心目标**
   - 访问参数
   - 初始化参数
   - 共享参数
   - 保存/加载参数

3. **关键机制**
   - 嵌套模型会**递归收集所有参数**
   - 共享参数 = 多个层引用同一个 `Parameter`
   - 梯度会**累加**到同一组参数

---

# 二、参数管理方法（完整列表）
## 1. 访问参数
- `net.state_dict()`：返回参数字典（最常用）
- `net.parameters()`：迭代所有参数
- `net.named_parameters()`：带名字的参数
- `net.children()` / `net.named_children()`：直接子模块
- `net.modules()` / `net.named_modules()`：递归所有模块

## 2. 查看参数信息
- `param.data`：参数数值
- `param.grad`：梯度
- `param.size()` / `shape`：形状
- `param.device`：在 CPU/GPU

## 3. 初始化参数
- 零初始化：`nn.init.zeros_()`
- 常数初始化：`nn.init.constant_()`
- 正态分布：`nn.init.normal_()`
- 均匀分布：`nn.init.uniform_()`
- Xavier：`nn.init.xavier_uniform_()`
- He 初始化：`nn.init.kaiming_normal_()`

## 4. 修改与绑定参数
- 直接赋值：`layer.weight.data.copy_(...)`
- 共享参数：多个层使用**同一个 Parameter 对象**
- 冻结参数：`param.requires_grad = False`

## 5. 设备与保存
- `net.to(device)`：参数移到 GPU/CPU
- `torch.save(net.state_dict(), path)`：保存参数
- `net.load_state_dict(torch.load(path))`：加载参数

# 三、课后习题 + 完整答案

## 5.1 模型构造 课后题答案

### 1. 如果将 MySequential 中代码改为使用 Python 列表，会发生什么？
**答案：**
会出现严重问题：
- 模型无法识别列表里的层为子模块
- `parameters()` 无法收集参数
- `to(device)` 不能把层搬到 GPU
- 模型保存、加载、初始化都会失效
本质：只有被注册为 `Module` 的属性，框架才会管理。

### 2. 实现一个块，它包含两个全连接层，并在前向传播时输出通道拼接
**答案：**
```python
class ParallelMLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(20, 256)
        self.fc2 = nn.Linear(20, 256)

    def forward(self, x):
        y1 = F.relu(self.fc1(x))
        y2 = F.relu(self.fc2(x))
        return torch.cat([y1, y2], dim=1)
```

### 3. 构建一个函数，生成许多相同的块，并用 Sequential 组合
**答案：**
```python
def make_blocks(num_blocks):
    layers = []
    for _ in range(num_blocks):
        layers.append(nn.Linear(64, 64))
        layers.append(nn.ReLU())
    return nn.Sequential(*layers)
```

## 5.2 参数管理 课后题答案

### 1. 查阅相关文档，查看 nn.init 模块提供了哪些不同的参数初始化方法
**答案：**
- 常数：`constant_`、`ones_`、`zeros_`
- 均匀：`uniform_`
- 正态：`normal_`
- 正交：`orthogonal_`
- Xavier / Glorot：`xavier_uniform_`、`xavier_normal_`
- He / Kaiming：`kaiming_uniform_`、`kaiming_normal_`
- 稀疏初始化：`sparse_`

### 2. 编写代码给网络初始化时使用不同的初始化
**答案：**
```python
def init_weights(m):
    if isinstance(m, nn.Linear):
        nn.init.xavier_uniform_(m.weight)
        nn.init.zeros_(m.bias)

net.apply(init_weights)
```

### 3. 在共享参数的层中，参数更新时梯度会发生什么变化？
**答案：**
共享参数时，多个位置的梯度会**累加**到同一个参数上。
因为它们本质是同一个对象，反向传播时所有路径的梯度相加。

### 4. 为什么共享参数是一个好主意？
**答案：**
- 大幅**减少参数量**
- 降低过拟合风险
- 强制提取相同特征（如 Siamese 网络、权重共享 CNN）
- 减少计算与内存开销

### 5. 尝试访问 `FixedHiddenMLP` 中的参数
**答案：**
```python
net = FixedHiddenMLP()
print(net.linear.weight)
print(net.linear.bias)
print(net.rand_weight)  # 常量权重不是 Parameter，不会被训练
```

# 四、面试题 + 标准答案（可直接背诵）

## 模型构造 面试题
### 1. nn.Module 是什么？为什么要继承它？
**答：**
`nn.Module` 是 PyTorch 所有网络模块的基类，提供参数管理、前向传播、设备迁移、保存加载等功能。
继承它可以自动管理参数，支持 GPU、反向传播、模型嵌套。

### 2. Sequential 的优缺点？
**答：**
优点：简单、快速搭建串行网络。
缺点：不支持复杂前向逻辑（if/for/多分支），灵活性差。

### 3. 自定义块时 forward 函数有什么用？
**答：**
定义数据如何流经网络，支持任意计算逻辑、控制流、层复用。

### 4. 为什么不能用 list 存层？
**答：**
list 中的层不会被注册为子模块，参数无法被框架管理，导致无法训练、无法迁移设备、无法保存。

---

## 参数管理 面试题
### 1. 如何查看模型所有参数？
**答：**
- `net.parameters()`：迭代参数
- `net.named_parameters()`：带名字
- `net.state_dict()`：字典形式（最常用）

### 2. 如何初始化参数？
**答：**
使用 `nn.init` 系列函数，通过 `net.apply(init_func)` 递归初始化所有层。

### 3. 什么是参数共享？有什么用？
**答：**
多个层共用同一组参数。
好处：减少参数量、防止过拟合、强制特征共享，常用于 CNN、Siamese 网络、RNN。

### 4. 如何固定某些层不训练？
**答：**
设置 `param.requires_grad = False`，
或 `layer.eval()`，并在优化器中只传入需要更新的参数。

### 5. state_dict 里存的是什么？
**答：**
只存**可学习参数和缓存（如 BN 的均值方差）**，不存模型结构，轻量、便于保存迁移。

# 一、5.3 读写文件（模型保存与加载）
## 核心知识点
1. **保存/加载 Tensor**
   - 保存：`torch.save(x, 'x.pt')`
   - 加载：`torch.load('x.pt')`

2. **保存/加载模型参数（最常用）**
   - 只存参数：`torch.save(net.state_dict(), 'path.pth')`
   - 加载：`net.load_state_dict(torch.load('path.pth'))`

3. **state_dict 是什么**
   - 字典：key=层名，value=参数张量
   - 只存**可学习参数 + BN 运行均值/方差**
   - 不存模型结构，体积小、易迁移

4. **保存整个模型（不推荐）**
   - `torch.save(net, 'net.pth')`
   - 缺点：依赖代码结构，跨版本/跨设备易报错

5. **加载时的常见问题**
   - 层名不匹配 → 加载失败
   - 训练/测试模式不一致 → 结果不对
   - 设备不匹配（CPU/GPU）→ 需要 map_location

---

# 二、5.4 使用GPU（CUDA）
## 核心知识点
1. **设备查询**
   - `torch.cuda.is_available()`：是否有GPU
   - `torch.cuda.device_count()`：GPU数量
   - `torch.device('cuda')` / `torch.device('cpu')`

2. **Tensor 移到GPU**
   - `x = x.cuda()`
   - `x = x.to('cuda')`

3. **模型移到GPU**
   - `net = net.to(device)`
   - 会**自动把所有参数移到对应设备**

4. **关键规则**
   - **Tensor 必须在同一设备才能运算**
   - CPU与GPU张量不能直接相加
   - 数据、模型、标签必须都在 cuda 上

5. **多GPU**
   - `nn.DataParallel` 简单并行
   - 实际训练常用 `DistributedDataParallel`

# 三、5.3 读写文件 课后题答案
### 1. 如果不实例化模型，直接加载 state_dict 会发生什么？
**答案：**
会报错。
必须先创建和原模型**结构完全一样**的模型实例，才能用 `load_state_dict` 加载参数。

### 2. 尝试只保存模型权重，再加载到结构相同但初始化不同的模型中
**答案：**
可以成功加载，并且**完全覆盖原有参数**。
结构一致 → 层名一致 → 可以正常加载。

### 3. 尝试加载结构不匹配的模型
**答案：**
会报错：
- Missing key(s) in state_dict
- Unexpected key(s) in state_dict
解决：严格保证模型结构一致，或手动过滤字典。

### 4. 保存整个模型 vs 保存 state_dict，哪个更好？为什么？
**答案：**
**state_dict 更好**
- 体积小
- 不依赖代码结构，兼容性强
- 迁移学习、微调更方便
- 避免序列化错误、版本不兼容

# 四、5.4 使用GPU 课后题答案
### 1. 尝试把一个在CPU上的张量和一个在GPU上的张量相加
**答案：**
直接报错：
`RuntimeError: Expected all tensors to be on the same device`
必须先把其中一个移到另一设备。

### 2. 将模型移到GPU，再打印参数，查看设备是否变化
**答案：**
参数设备会从 `cpu` 变为 `cuda:0`。
`net.to(device)` 会递归移动所有参数。

### 3. 用两个GPU运行计算
**答案：**
```python
net = nn.DataParallel(net, device_ids=[0,1])
```
数据会自动分到多卡，结果汇总。

### 4. 查看显卡显存占用
**答案：**
- 命令行：`nvidia-smi`
- PyTorch：`torch.cuda.memory_allocated()`

# 五、面试题 + 标准答案（可直接背诵）
## 5.3 读写文件 面试题
### 1. 为什么推荐保存 state_dict 而不是整个模型？
**答：**
- state_dict 只存参数，体积小、加载快
- 不依赖模型代码结构，兼容性强
- 方便微调、迁移学习、部分加载
- 避免序列化错误、跨版本问题

### 2. 加载模型时出现 key 不匹配怎么办？
**答：**
- 检查模型结构是否一致
- 检查层名是否变化
- 可通过 `strict=False` 忽略不匹配
- 或手动过滤 state_dict 字典

### 3. 训练好的模型如何在CPU上加载？
**答：**
`torch.load(path, map_location='cpu')`

---

## 5.4 GPU 使用 面试题
### 1. 为什么会报 device 不一致错误？
**答：**
PyTorch 不允许 CPU 张量与 GPU 张量直接运算，必须在同一设备。

### 2. to('cuda') 是原地操作吗？
**答：**
不是原地操作，需要重新赋值：
`x = x.to('cuda')`
模型可以 `net.to('cuda')` 因为 Module 内部会修改。

### 3. 模型、数据、标签都要移到GPU吗？
**答：**
必须全部移到同一GPU，否则前向传播报错。

### 4. DataParallel 有什么缺点？
**答：**
- 负载不均衡
- 单进程多线程，效率不如 DDP
- 不适合大规模训练

### 5. CUDA out of memory 怎么解决？
**答：**
- 减小 batch size
- 降低图像尺寸
- 用混合精度训练
- 释放无用张量
- 检查是否累积计算图
