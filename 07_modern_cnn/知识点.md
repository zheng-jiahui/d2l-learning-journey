# 1×1卷积 = 全连接层
---
### 原理
**输入**：`(H, W, C_in)`  
**1×1卷积**：`(C_in, C_out)` 的权重矩阵

对每个像素位置 `(h, w)`：
- 取出该位置的 `C_in` 个值
- 与权重矩阵做**矩阵乘法**
- 输出 `C_out` 个值

这和全连接层**完全一样**：输入 `C_in` 维，输出 `C_out` 维

---

### 区别

| 维度 | 1×1卷积 | 全连接 |
|------|---------|--------|
| 计算方式 | 逐像素共享权重 | 全局展平后计算 |
| 空间信息 | **保持** H×W | **丢失** 空间结构 |
| 权重共享 | 是（所有位置用同一组权重） | 否 |

---

### 本质

**1×1卷积 = 在空间每个位置独立做全连接**

- 全连接：一次处理整个特征图 → 输出向量
- 1×1卷积：每个像素位置独立做全连接 → 输出特征图


## NiN核心思想

**用全局平均池化 + 1×1卷积 替代全连接层**

---

### 传统CNN（AlexNet）
```
卷积层 → 卷积层 → ... → 全连接层 → 输出
```
- 全连接层参数巨大（AlexNet最后3层占90%参数）
- 容易过拟合

---

### NiN做法

```
卷积层 → NiN块 → 卷积层 → NiN块 → ... → 全局平均池化 → 输出
```

**最后的结构**：
1. 最后一个NiN块的输出通道数 = 类别数（如10）
2. 全局平均池化：每个特征图求平均 → 得到10个值
3. 直接作为分类结果

---

### 举例

假设分类10类，最后特征图是 `(10, H, W)`：
- 全局平均池化：对每个通道求平均 → `(10,)`
- 第i个值 = 第i类的得分
- **不需要全连接层**

---

### 为什么可以这样？

**1×1卷积** 在每个位置做**跨通道信息融合**，相当于：
- 最后一层用1×1卷积把通道数压缩到类别数
- 每个通道代表一个类别的响应
- 全局平均池化得到整张图对该类别的响应强度

---

### 优势

1. **参数极少**：无全连接层
2. **防过拟合**：强制特征图对类别有空间响应
3. **可解释性**：每个通道可视化对应类别的激活区域


## 1×1卷积的三大作用

---

### 1. 降维/升维

**改变通道数**，不改变空间尺寸

```python
# 输入 (H, W, 256)
nn.Conv2d(256, 64, kernel_size=1)  # 降到64通道
nn.Conv2d(64, 256, kernel_size=1)  # 升回256通道
```

**意义**：减少计算量（先在低维做卷积，再升回去）

---

### 2. 跨通道信息融合

每个1×1卷积核：对同一位置的C_in个通道做**加权组合**

```
输出通道0 = w0×通道0 + w1×通道1 + ... + w_{C_in-1}×通道_{C_in-1}
```

**意义**：让不同通道的信息交互，比3×3卷积更高效

---

### 3. 增加非线性

```python
nn.Conv2d(in, out, kernel_size=1)
nn.ReLU()
```

1×1卷积 + ReLU = 对每个像素位置做**非线性变换**

**意义**：增加网络深度，提升表达能力，几乎不增加参数量

---

## 典型应用

| 网络 | 用途 |
|------|------|
| NiN | 替代全连接层 |
| GoogLeNet | Inception内降维，减少计算 |
| ResNet | 瓶颈层：256→64→256 |
| MobileNet | 深度卷积后升维 |

---

## 计算量对比

假设输入 `(H, W, 256)`，输出 `(H, W, 64)`

- **3×3卷积**：`3×3×256×64 × H×W = 147456 × H×W`
- **1×1卷积**：`1×1×256×64 × H×W = 16384 × H×W`

1×1卷积计算量是3×3的 **1/9**


**是的，标准顺序是：**

```
卷积 → 激活(ReLU) → 池化
```

---

## 原因

1. **卷积**：提取特征
2. **ReLU**：引入非线性，让网络能学习复杂模式
3. **池化**：降维，保留主要特征

---

## 例外情况

- **池化后也可以加ReLU**（但通常不需要，因为池化不改变值的正负）
- **残差网络**：顺序不同（卷积→BN→ReLU→卷积→BN→加→ReLU）

---

**标准CNN顺序记住**：`Conv → BN(可选) → ReLU → Pool`

## Inception核心思想

**同一层用多个不同尺寸的卷积核并行提取特征，然后拼接**

---

### 结构

```
输入
  ├─ 1×1卷积 ──┐
  ├─ 3×3卷积 ──┤
  ├─ 5×5卷积 ──┼─ 拼接 → 输出
  └─ 3×3池化 ──┘
```

---

### 为什么这样做

| 问题 | Inception方案 |
|------|---------------|
| 不知道用多大卷积核 | 全用，让网络自己学 |
| 不同物体尺寸不同 | 多尺度同时提取 |
| 计算量太大 | 1×1卷积先降维 |

---

### 降维版Inception

```
输入
  ├─ 1×1 ──────────────┐
  ├─ 1×1 → 3×3 ────────┤
  ├─ 1×1 → 5×5 ────────┤
  └─ 3×3池化 → 1×1 ────┘
```

**1×1降维**：先减少通道数，再做大卷积，计算量降低 **4-5倍**

---

### 拼接规则

所有分支输出**尺寸相同**（H,W相同），在**通道维**拼接

```
分支1: (H,W,64)
分支2: (H,W,128)
分支3: (H,W,32)
分支4: (H,W,32)
输出: (H,W,256)
```

---

### 思想总结

1. **多尺度**：同时看细节（1×1）和上下文（5×5）
2. **稀疏连接**：1×1降维让大卷积变稀疏
3. **模块化**：复杂网络由Inception块堆叠

**一句话**：不知道哪种卷积好，就全用，让网络自己选


# 7.5 批量规范化
## 一、核心重点（一句话吃透）
批量规范化（BN）是对网络**中间层输入**做**小批量均值方差归一化**+**可学习缩放偏移**，**训练/预测行为不同**，核心作用是**稳分布、加速收敛、缓解梯度消失/爆炸、自带弱正则**。

---
## 二、必记知识点（面试/考试必背）
### 1. 核心公式
\[
\mathrm{BN}(x)=\gamma \odot \frac{x-\hat{\mu}_B}{\sqrt{\hat{\sigma}_B^2+\epsilon}}+\beta
\]
- \(\hat{\mu}_B\)：小批量均值
- \(\hat{\sigma}_B^2\)：小批量方差
- \(\epsilon\)：防除0（常用1e-5）
- \(\gamma\)（缩放）、\(\beta\)（偏移）：可学习参数
- 训练用**批量统计**，预测用**移动平均统计**。

### 2. 训练 vs 预测
- 训练：用当前batch算均值/方差，同步更新**moving_mean/moving_var**
- 预测：直接用训练好的移动均值/方差，无batch噪声，输出确定。

### 3. 放置位置
- 全连接/卷积层**之后**，**激活函数之前**
- 卷积：按**通道**归一化，每个通道一套\(\gamma/\beta\)
- 全连接：按**特征维度**归一化。

### 4. 关键作用
1. 稳定中间层分布，抑制**内部协变量偏移**
2. 允许**更高学习率**，大幅加速收敛
3. 缓解梯度消失/爆炸，支持训练深层网络
4. 自带**弱正则**，可减少Dropout依赖
5. 对初始化不敏感。

### 5. 重要细节
- BatchSize不能太小（建议≥32），太小统计不准
- BatchSize是我训练batch的大小，而\(\gamma/\beta\)可以学习
- 卷积用**BatchNorm2d**，全连接用**BatchNorm1d**
- 加BN后，前面层的**偏置b可省略**（被\(\beta\)覆盖）。

---
## 三、课后题简要答案
1. **可删卷积/全连接层偏置吗？**
可以。BN的\(\beta\)已承担偏移，原偏置冗余，删后不影响性能并减少参数量。
2. **BN对学习率的影响**
用BN可设**更大学习率**（如LeNet从0.1→1.0），收敛更快、精度更稳。
3. **是否每层都要BN？**
不必。关键层（深层、特征变化大的层）加即可，过多会增加计算。
4. **BN可替代Dropout吗？**
可部分替代。BN有正则效果，但复杂任务常搭配使用，效果更稳。
5. **\(\gamma/\beta\)作用**
\(\gamma\)恢复方差，\(\beta\)平移均值，让网络保留表达能力。

---
## 四、面试高频题（标准答案）
### 1. BN为什么能加速训练？
稳定各层输入分布，避免激活进入饱和区，梯度更大更稳，支持高学习率，收敛更快。

### 2. 训练和预测时BN有何不同？
训练用**当前batch**统计并更新移动平均；预测用**固定移动平均**，无batch噪声，单样本也可预测。

### 3. 为什么BN有正则作用？
batch均值/方差含噪声，类似轻微扰动，减少过拟合；大batch正则弱，小batch正则强。

### 4. BN、LN、IN、GN区别？
- BN：按**batch+通道**归一化，CV常用，小batch不稳
- LN：按**样本+特征**归一化，NLP常用，不受batch影响
- IN：按**单样本单通道**，风格迁移
- GN：按**分组通道**，小batch下比BN稳。

### 5. 加BN后为什么可以删偏置b？
归一化会减去均值，偏置b被抵消，BN的\(\beta\)已提供可学习偏移，保留b冗余无增益。

### 6. BN真的解决内部协变量偏移吗？
学界有争议。实验表明BN核心是**让优化曲面更平滑**，而非单纯抑制协变量偏移，但工程上仍按稳分布理解即可。


**是，传入的是** **`num_features`** **= 通道数**


## BN使用对照表

| 层类型 | BN类 | 传入参数 | 输入形状 | 归一化维度 |
|--------|------|----------|----------|------------|
| 卷积 | `BatchNorm2d` | `out_channels` | (batch, C, H, W) | 每个通道 |
| 全连接 | `BatchNorm1d` | `out_features` | (batch, features) | 每个特征维 |

---

## 代码示例

```python
# 卷积
nn.Conv2d(1, 64, kernel_size=3)
nn.BatchNorm2d(64)
nn.ReLU()

# 全连接
nn.Linear(784, 256)
nn.BatchNorm1d(256)
nn.ReLU()
```

**BatchNorm解决深层网络训练不稳定的问题**

---

## 三个问题

1. **梯度消失/爆炸**：层数越深，梯度越不稳定
2. **内部协变量偏移**：每层输入分布随前层变化而变化
3. **对初始化敏感**：初始值不好，网络难收敛

---

## BN解决什么

| 问题 | BN的作用 |
|------|----------|
| 梯度不稳定 | 保持每层输出在合理范围 |
| 分布变化 | 固定每层输入的分布 |
| 初始化敏感 | 降低对初始值的依赖 |

---

## 效果

- 可以用更大学习率
- 训练更快
- 轻微正则化
- 减少Dropout需求


# 7.6 ResNet 核心速记
## 一、核心重点（一句话）
ResNet 通过**残差连接**让网络拟合**残差映射 f(x)−x**，而非直接拟合 f(x)，轻松训练极深层网络，解决**深度网络退化**问题。

---
## 二、必记知识点
### 1. 核心思想
- 普通网络：加深可能**退化**（准确率下降）
- ResNet：让新增层拟合**恒等映射 x**，保证加深不会变差
- 公式：**输出 = 主分支卷积结果 + 捷径输入 x**

### 2. 残差块结构
- 2 个 3×3 卷积 + BN + ReLU
- 捷径连接（skip connection）：直接把输入加到输出
- 通道/尺寸不匹配时：用 **1×1 卷积 + stride=2** 对齐

### 3. 关键规则（必背）
1. **每个 stage 第 0 个残差块**：下采样（stride=2）+ 通道翻倍
2. **第一个 stage 不下采样**（first_block=True）
3. 卷积→BN→ReLU，最后相加再 ReLU
4. 下采样只在**每个 stage 开头做一次**

### 4. ResNet-18 结构
1. 7×7 卷积 + BN + ReLU + 最大池化
2. 4 个 stage：64→128→256→512 通道
3. 每个 stage 含 2 个残差块
4. 全局平均池化 + 全连接输出

### 5. 为什么强
- 梯度可通过捷径直达浅层，**缓解梯度消失**
- 优化更简单，深层易训练
- 泛化强，成为 CV  backbone 基础

---
## 三、课后题简要答案
1. **Inception 与残差块区别**
Inception：多尺度卷积**通道拼接**；残差块：**元素相加**，核心是拟合残差。
2. **ResNet 瓶颈结构（Bottleneck）**
1×1（降维）→3×3→1×1（升维），减少计算，适合 ResNet50/101/152。
3. **BN−ReLU−Conv 改进**
先归一化激活再卷积，提升稳定与精度。
4. **嵌套函数仍要限制复杂度**
过深会**过拟合、计算爆炸、优化困难**，并非越深越好。

---
## 四、面试高频题（标准答案）
### 1. ResNet 解决什么问题？
深度网络**退化**（层数加深准确率不升反降），不是过拟合，是优化难题。

### 2. 残差连接为什么有效？
- 拟合残差比拟合原始映射更容易
- 梯度可直通浅层，**缓解梯度消失**
- 网络可轻松学到恒等映射，加深不退化

### 3. 什么时候用 1×1 卷积？
- 通道数改变
- 特征图宽高需要减半（stride=2）
- 保证主分支与捷径**形状完全一致**才能相加

### 4. 训练和预测时残差块有区别吗？
无区别；BN 在训练/预测不同，但残差加法逻辑不变。

### 5. ResNet 为什么比 VGG/GoogLeNet 深得多？
残差连接让梯度传播更顺畅，可轻松训练上百层。

### 6. ResNet 与 DenseNet 区别？
ResNet：**逐块相加**；DenseNet：**所有前面层特征拼接**。


**残差快解决深层网络退化问题**

---

## 退化问题

层数越深，训练误差反而越高（不是过拟合）

- 56层网络比20层网络误差更大
- 理论上更深网络应该表现更好
- 实际：梯度消失/爆炸 + 优化困难

---

## 残差块解决

```
输出 = F(x) + x
```
- 恒等映射：如果F(x)学不好，至少保留x
- 深层至少不比浅层差
- 梯度可以直接回传（跳过中间层）

---

## 效果

- 可以训练1000+层
- 梯度不消失
- 成为现代CNN标配


## DenseNet 一句话总结

**每层与前面所有层在通道维拼接，实现极致特征重用**

---

## 核心公式

```
x_l = [x_0, x_1, ..., x_{l-1}, F_l(x_0,...,x_{l-1})]
```

---

## 与ResNet区别

| | ResNet | DenseNet |
|--|--------|----------|
| 操作 | 相加 `+` | 拼接 `concat` |
| 梯度 | 好 | 极好 |
| 参数 | 少 | 更少 |
| 显存 | 正常 | 大（要存所有特征图）|

---

## 缺点

显存占用大，实际用得少

---

**结论**：知道这个思想就行，不用深究
